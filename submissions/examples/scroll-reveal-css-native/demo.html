scroll-reveal-css-native

Scroll-triggered entrance animation using animation-timeline: view() — zero JavaScript.

↓ Scroll down ↓
Example 1 — Default (fade + slide up)

This heading fades up as you scroll.

Add .scroll-reveal to any element. It sits invisible below the viewport and animates into view as you scroll it up — driven entirely by your scroll position, no JavaScript needed.

Uses animation-timeline: view() and animation-range: entry 0% entry 100% so the animation progress is tied directly to how far the element has entered the viewport.

Example 2 — Cards (each animates independently)

Each card has its own scroll timeline.

Because every element gets its own view() timeline, cards stagger naturally as you scroll — no delay utilities needed.

Example 4 — Fade only

No movement, just opacity.

.scroll-reveal-fade transitions opacity only — no translate — perfect for full-width images or banners where movement would look off.

Example 5 — Slow variant

A more dramatic, slower entrance.

.scroll-reveal-slow uses a larger translate offset so the animation travels further — best for hero sections or large feature blocks.

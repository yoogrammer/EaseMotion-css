❌ Bug

The Problem — top: 0 is hardcoded

.ease-sticky currently sets position: sticky; top: 0. On any page with a fixed navbar, a sticky sidebar or section header will hide under the navbar. There is no way to change the offset using only utility classes — developers must write custom CSS to override the class. 

/* current core/utilities.css line 161 */
.ease-sticky { position: sticky; top: 0; }
✅ Fix

One-line fix — CSS custom property

Replace the hardcoded 0 with a CSS custom property that falls back to 0. All existing uses are completely unaffected. 

/* BEFORE */
.ease-sticky { position: sticky; top: 0; }

/* AFTER */
.ease-sticky { position: sticky; top: var(--ease-sticky-top, 0); }

Optionally, add --ease-sticky-top: 0; to core/variables.css for discoverability in the token reference.

Usage after fix

<!-- Default: sticks at 0 (unchanged) -->
<div class="ease-sticky">Sticks at top: 0</div>

<!-- Inline override: clears a 56px navbar -->
<div class="ease-sticky" style="--ease-sticky-top: 56px;">
  Sidebar
</div>

<!-- Layout-level override: all sticky children inherit it -->
<main style="--ease-sticky-top: 4rem;">
  <aside class="ease-sticky">Sidebar</aside>
</main>

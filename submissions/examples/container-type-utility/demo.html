Compare: size vs. inline-size

.container-type-inline-size

Inline only

Responds only to width changes. Height changes are ignored.

.container-type-size

Both axes

Responds to width and height changes. Layout may be more expensive.

.container-type-normal

No query

No containment context — @container queries will not match.

Tip: container-type: inline-size is the most common choice. It creates a containment context only in the inline (text) direction, so the element can still grow vertically with its content. Use container-type: size only when you need height-based queries.

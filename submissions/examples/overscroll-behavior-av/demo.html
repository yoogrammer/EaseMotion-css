overscroll-behavior Refactor

This page has a very tall body (simulating a long article). The sidebar on the left also has internal scrolling because it contains many items.

Because the sidebar utilizes the modern overscroll-behavior: contain property, if you scroll to the absolute bottom of the sidebar, the scroll momentum gets safely "trapped". It will not bubble up and accidentally scroll the main page, nor will it trigger a mobile browser's destructive "Pull-to-Refresh" action!

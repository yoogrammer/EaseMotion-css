Frequently Asked Questions

Everything you need to know about EaseMotion CSS

What is EaseMotion CSS?

EaseMotion CSS is a zero-dependency, animation-first CSS framework that lets you build polished interfaces with readable utility class names like ease-fade-in and ease-slide-up. No build step is required — just link one stylesheet.

Do I need a build tool to use it?

No. EaseMotion CSS is designed to work by linking a single CSS file via CDN or npm. You don't need Webpack, Vite, or any other build configuration. Just paste the <link> tag and start using classes.

How is it different from Tailwind CSS?

EaseMotion CSS uses human-readable class names that describe behavior (ease-center, ease-hover-grow) rather than short utility shorthands (flex items-center). It also treats animation as a first-class feature and requires no build tool or configuration file.

Can I use EaseMotion CSS with React or Vue?

Yes. EaseMotion CSS works with any framework that renders standard HTML class attributes. Install from npm with npm install easemotion-css, then import the stylesheet once and use the classes in your components.

How do I customize the colors and timing?

EaseMotion CSS exposes 60+ CSS custom properties. Override any of them in your own stylesheet without using !important. For example: :root { --ease-color-primary: #f97316; --ease-speed-medium: 400ms; }

Is there a dark mode?

A dedicated dark mode token layer is on the roadmap for v1.1. In the meantime, you can override CSS custom properties inside a [data-theme="dark"] selector or using @media (prefers-color-scheme: dark) to create your own dark theme.

Tabs: Outline Offset & Focus

Negative outline-offset clips keyboard focus indicators, creating an accessibility violation.

Before: Negative outline-offset

Uses outline-offset: -4px which clips the focus ring inside the tab button, making it hard to see.

Use Tab to focus the tabs and look for the focus ring. With negative outline-offset, the ring is clipped and barely visible.

Focus

The outline is clipped inside the border (barely visible).

.ease-tab:focus-visible {
  outline: 2px solid currentColor;
  outline-offset: -4px;
}
Focus ring clipped

After: Positive outline-offset

Uses outline-offset: 2px with a visible primary color. The focus ring is clearly visible outside the element.

Use Tab to focus the tabs. The focus ring is clearly visible outside each tab button.

Focus

The outline sits outside the element — clearly visible.

.ease-tab:focus-visible {
  outline: 2px solid var(--ease-color-primary, #6366f1);
  outline-offset: 2px;
}
Focus ring visible

Accessibility impact

The WCAG 2.4.7 (Focus Visible) requires that keyboard focus indicators be clearly visible. A clipped or invisible focus ring fails this criterion.

  • Minimum: Focus indicator must be at least as large as a 2px outline
  • Enhanced: Focus indicator should have contrast ratio of at least 3:1 against adjacent colors
  • Never use negative outline-offset on interactive elements

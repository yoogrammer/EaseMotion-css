ease-scroll-indicator

A reading progress bar at the top of the page — scroll down to see it fill

By EaseMotion CSS · 5 min read · June 5, 2026

This is a long-form article to demonstrate the scroll indicator component. As you scroll down the page, the thin gradient bar at the very top of the viewport fills from left to right, showing your reading progress from 0% to 100%.

The component uses a CSS-only approach via animation-timeline: scroll() in supported browsers, with a JavaScript scroll event listener as fallback for broader compatibility.

How it works

The scroll indicator is a fixed-position element at the top of the viewport. Its width is tied to the scroll progress of the page. In modern browsers (Chrome 115+), the CSS animation-timeline: scroll() property drives the animation natively without any JavaScript.

For browsers that don't support scroll-driven animations yet, a lightweight JavaScript fallback listens to the scroll event and updates the bar width accordingly. The bar is hidden when the page content fits entirely within the viewport.

Variants

The component supports several variants out of the box. Use .ease-scroll-indicator--solid for a solid colour instead of gradient. Use .ease-scroll-indicator--thin (2px) or .ease-scroll-indicator--thick (6px) to adjust the bar height. Custom colours can be set via the --ease-scroll-color CSS variable.

The bar respects prefers-reduced-motion by showing a static full-width bar, and adapts to dark mode automatically via the prefers-color-scheme media query.

Use cases

Reading progress bars are commonly seen on blog posts, documentation pages, long-form articles, tutorials, and any content-heavy page where users benefit from understanding how much content remains. GitHub uses a similar pattern on README files.

The indicator is intentionally thin and unobtrusive — it sits at the very top edge of the viewport without interfering with navigation bars, headers, or other UI elements. Its high z-index (--ease-z-toast: 9999) ensures it always renders on top.

Usage

Simply add the element to your page and the JS snippet will handle the rest. The CSS animation provides progressive enhancement for modern browsers, while the JS ensures consistent behaviour everywhere.

Scroll-driven animations are part of the CSS Scroll-Driven Animations specification, which is actively being developed. As browser support grows, the JS fallback will become unnecessary.

Conclusion

The ease-scroll-indicator is a lightweight, accessible, and progressively-enhanced reading progress bar that fits seamlessly into the EaseMotion CSS component library. It works out of the box with minimal markup and zero configuration.

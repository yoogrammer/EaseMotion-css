Container Query Units Deep Dive

Resize the container below to see cqw / cqh / cqi / cqb / cqmin / cqmax in action. Typography and spacing respond to the container's size, not the viewport.

cqw

Width Relative

This title uses font-size: 5cqw — 5% of the container width. The badge value updates dynamically via CSS counter and the attr() of a data attribute set by JS.

5cqw = 30px
cqh

Height Relative

Height-based sizing: padding: 2cqh and font-size: 4cqh. Useful when container height is well-defined, such as in cards inside a full-height grid.

4cqh ≈ 20px
cqi

Inline Size

cqi refers to the container's inline size (width in horizontal writing modes). The gap between cards uses gap: 3cqi.

gap: 3cqi
cqb

Block Size

cqb references the container's block axis (height in horizontal mode). We use margin-block: 2cqb for vertical rhythm.

margin: 2cqb
cqmin

Min Axis

cqmin equals the smaller of cqi and cqb. The border-radius here is 1cqmin — it shrinks gracefully on the smaller axis.

radius: 1cqmin
cqmax

Max Axis

cqmax equals the larger of cqi and cqb. The box-shadow offset uses 0.5cqmax for an exaggerated effect on the larger axis.

shadow: 0.5cqmax

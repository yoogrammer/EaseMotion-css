Button color-mix() @supports Fallback

color-mix() in hover states breaks in older browsers without a proper @supports wrapper.

Issue #5754

The Problem

color-mix(in srgb, var(--ease-btn-bg), black 20%) is used in button hover states without @supports. Safari < 16.2, older Firefox, and some mobile browsers do not support color-mix() — the button color disappears entirely.

⚠️ Browser support note: If you're viewing this in a modern browser, both buttons will work. The broken button simulates the fallback failure by omitting @supports. In an actual unsupported browser, the broken button would show no hover color change.

Primary Buttons

Broken — No @supports

color-mix() unwrapped

.btn { background: #3b82f6; }
.btn:hover {
background: color-mix(in srgb, #3b82f6, black 20%);
}
Fixed — With @supports

color-mix() with fallback

.btn { background: #2563eb; }
.btn:hover { background: #1d4ed8; }

@supports (color: color-mix(in srgb, red, blue)) {
.btn:hover {
background: color-mix(in srgb, #2563eb, black 20%);
}
}

Outline Buttons

Broken — No @supports

color-mix() unwrapped

.btn-outline:hover {
border-color: color-mix(...);
color: color-mix(...);
}
Fixed — With @supports

color-mix() with fallback

.btn-outline:hover {
border-color: #1d4ed8;
color: #1d4ed8;
}

@supports (color: color-mix(...)) {
.btn-outline:hover { ... }
}

The Fix Pattern

Always wrap color-mix() in @supports

/* Step 1: Solid color fallback (works everywhere) */
.ease-btn-primary:hover {
background: #1d4ed8;
}

/* Step 2: Enhanced color-mix for modern browsers */
@supports (color: color-mix(in srgb, red, blue)) {
.ease-btn-primary:hover {
background: color-mix(in srgb, #2563eb, black 20%);
}
}

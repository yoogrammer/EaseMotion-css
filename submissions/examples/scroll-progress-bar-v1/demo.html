Scroll-Driven Top Progress Bar

Scroll down the page to see the top progress bar expand in perfect sync with your viewport position. Powered by CSS Scroll Timelines.

1. Pure CSS Scroll Timeline

This implementation utilizes the native CSS animation-timeline: scroll(root block) spec. As you scroll down the document, the browser automatically adjusts the animation's current time index based on the root scrollbar percentage. This means:

  • Zero JavaScript: No event listeners, no scroll thrashing, and zero main-thread overhead.
  • Sub-pixel Accuracy: Rendering is managed directly by the browser's compositor layer.
  • Instant Load: Works instantly upon HTML load without waiting for JS execution.

2. Safe Degradation

For browsers that do not yet support CSS scroll-driven animations, a fallback is defined via the @supports not (animation-timeline: scroll()) { } block. In non-supporting engines, the progress bar remains at a constant full-width configuration with subtle opacity, acting as a clean top highlight accent instead of rendering broken layout fragments.

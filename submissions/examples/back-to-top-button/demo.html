Button

A floating scroll-to-top button with an SVG progress ring, fade-in with spring physics, pulse on appear, and full CSS variable theming. Scroll down to see it in action.

Scroll down to trigger the button

Features

SVG Progress Ring

A circular SVG ring tracks how far you've scrolled down the page, giving users a clear sense of position.

Spring-physics Entrance

The button appears with a cubic-bezier overshoot curve and a pulse ring — not a plain fade.

CSS Variable Theming

Color, size, position, ring width — all controlled from :root with no JS changes needed.

Accessible & Reduced-Motion Safe

Proper aria-label, keyboard-focusable, and all animations disabled for users who prefer reduced motion.

Usage

<!-- 1. Add to <body> -->
<div class="em-btt-wrap" id="emBttWrap">
  <svg class="em-btt-svg" viewBox="0 0 52 52">
    <circle class="em-btt-track"    cx="26" cy="26" r="23"/>
    <circle class="em-btt-progress" cx="26" cy="26" r="23" id="emBttProgress"/>
  </svg>
  <button class="em-btt" id="emBtt" aria-label="Scroll back to top">
    <span class="em-btt-arrow"></span>
  </button>
</div>

<!-- 2. Link stylesheet & script -->
<link rel="stylesheet" href="style.css">
<script src="btt.js"></script>

<!-- 3. Optional: override theme -->
<style>
  :root { --em-btt-color: #0ea572; }
</style>

Why this fits EaseMotion CSS

EaseMotion CSS is animation-first. This component expresses every state — hidden, visible, hovered, active — purely through CSS transitions and keyframes. The JS is a thin scroll listener that only toggles classes and updates a single SVG attribute.

All visual customisation lives in CSS custom properties, keeping the component framework-agnostic and trivially themeable. It follows the em- naming convention and requires zero build tools.

Scroll position

Keep scrolling — the ring fills as you go deeper, and empties as you scroll back up.

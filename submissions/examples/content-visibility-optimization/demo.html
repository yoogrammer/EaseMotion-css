Content Visibility

Rendering performance with content-visibility and contain-intrinsic-size — no JavaScript required.

Visible Content (no optimization)

This content is always rendered — it is in the initial viewport. Without content-visibility, every element on the page is rendered immediately, even if it is far below the fold. For long pages with many sections, this can cause significant layout and paint costs on initial load.

Optimized Content

These sections use content-visibility: auto combined with contain-intrinsic-size. The browser skips rendering them until they are scrolled near the viewport. The intrinsic size placeholder prevents layout shift when the content eventually renders.

Section 1

content-visibility: auto tells the browser to skip this element's rendering (layout, paint, and compositing) until it is close to the viewport. This applies to the element itself and all its descendants.

Section 2

contain-intrinsic-size: 240px reserves the height of the element before it renders. Without it, the element would collapse to zero height while off-screen, causing a jarring layout shift when it scrolls into view and suddenly appears.

Section 3

Together, content-visibility: auto and contain-intrinsic-size provide a smooth lazy-rendering experience. The browser knows the size of the content before rendering it, and only paints it when the user scrolls nearby.

Section 4

This technique is especially useful for long lists, feeds, virtualized content, or any page with many sections below the fold. It can reduce initial render time by 50% or more on content-heavy pages.

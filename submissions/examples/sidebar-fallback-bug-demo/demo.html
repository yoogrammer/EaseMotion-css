Wrong Fallback Values in Sidebar

components/sidebar.css uses #111 for neutral-900 (should be #0f172a) and #e5e7eb for neutral-200 (should be #e2e8f0). If CSS variables fail, wrong colors render.

Bug Demo

☰ Dashboard
⚙ Settings

The sidebar renders with #111 background and #e5e7eb border — these are wrong fallbacks. With CSS variables working, the correct values (#0f172a / #e2e8f0) should appear.

The Bug

components/sidebar.css line 17: background: var(--ease-color-neutral-900, #111) — fallback #111 is not the real neutral-900 (#0f172a).

Line 18: border-right: 1px solid var(--ease-color-neutral-200, #e5e7eb) — fallback #e5e7eb is not the real neutral-200 (#e2e8f0).

The Fix (in this demo's style.css)

.ease-sidebar {
  background: var(--ease-color-neutral-900, #0f172a);
  border-right: 1px solid var(--ease-color-neutral-200, #e2e8f0);
}

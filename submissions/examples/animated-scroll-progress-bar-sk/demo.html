Scroll-driven progress

This page uses CSS animation-timeline: scroll() — a pure-CSS scroll-driven animation — to drive a thin progress bar at the top of the viewport. No JavaScript required.

Supported in Chromium browsers (Chrome 115+, Edge 115+).

What is scroll-driven animation?

CSS scroll-driven animations let you tie animation progress directly to a scroll container's scroll position. The scroll() timeline maps 0% (top of scroll) to 0% of the animation, and 100% (bottom) to 100%. This means you can build progress bars, parallax effects, and reveal animations with zero JavaScript.

Conclusion

Scroll-driven animations unlock a new category of interaction that previously required JavaScript and scroll event listeners. The performance is better (runs on the compositor thread) and the code is simpler. For browsers that don't yet support the feature, the page degrades gracefully — readers see all the content, just without the progress bar animation.

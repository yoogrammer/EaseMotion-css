Scroll Reveal Motion Guard

Scroll-driven reveal animations WITHOUT vs WITH a prefers-reduced-motion guard.

Before: No motion guard

The reveal animation plays for all users, including those who have requested reduced motion. This can cause discomfort for people with vestibular disorders.

.ease-scroll-reveal {
  animation: fadeSlideUp 0.6s;
  animation-timeline: view();
}

/* No reduced-motion guard */

After: With motion guard

A prefers-reduced-motion media query disables the animation for users who prefer reduced motion, while keeping the content visible.

.ease-scroll-reveal {
  animation: fadeSlideUp 0.6s;
  animation-timeline: view();
}

@media (prefers-reduced-motion: reduce) {
  .ease-scroll-reveal {
    animation: none;
    opacity: 1;
    transform: none;
  }
}

How to implement the fix

Two small additions to the scroll-reveal component:

  1. Wrap the animation in a @media (prefers-reduced-motion: no-preference) block
  2. Provide a static visible state as the default
.ease-scroll-reveal {
  opacity: 1; /* visible by default for reduced-motion users */
  transform: none;
}

@media (prefers-reduced-motion: no-preference) {
  .ease-scroll-reveal {
    opacity: 0;
    transform: translateY(30px);
    animation: fadeSlideUp 0.6s ease-out;
    animation-timeline: view();
  }
}

Bug Fix: .ease-btn-loading smoke test mismatch

The smoke test asserts font-size: 0 to hide button text in loading state, but buttons.css actually uses color: transparent !important. The test will always fail even though the component works correctly.

❌ What the test checks (smoke.test.js line 91)
expect(css).toContain('font-size: 0');
// This string does NOT exist in buttons.css
// → test always FAILS
✅ What the component actually does (buttons.css line 215)
.ease-btn-loading {
  color: transparent !important;
  /* text hidden via color, NOT font-size */
}

Live Loading Button (component works fine)

The actual button behaviour is correct — text is visually hidden, spinner shows. Only the test is wrong.

Current implementation — color: transparent
Alternative — font-size: 0 (if maintainer prefers this approach)

Proposed Fix (two options for the maintainer)

⭐ Option A — fix the test (recommended)

The component's color: transparent approach is semantically superior — it keeps text in the accessibility tree and font metrics intact so the spinner size is stable. The test should match the implementation. 

// tests/smoke.test.js — line 89-96
it('should hide plain text in loading buttons and keep the spinner visible', () => {
  expect(css).toContain('.ease-btn-loading');
- expect(css).toContain('font-size: 0');          // ← remove
+ expect(css).toContain('color: transparent');     // ← add
  expect(css).toContain('.ease-btn-loading > *');
  expect(css).toContain('visibility: hidden');
  expect(css).toContain('.ease-btn-loading::after');
  expect(css).toContain('border: 2px solid currentColor');
});
Option B — fix the component

If font-size: 0 was the original design intent, update buttons.css to use it. Note: this collapses the button's text node to zero size, which may affect the spinner's em-based dimensions. 

/* components/buttons.css — .ease-btn-loading */
.ease-btn-loading {
  pointer-events: none;
  position: relative;
- color: transparent !important;   /* ← remove */
+ font-size: 0;                    /* ← add    */
}

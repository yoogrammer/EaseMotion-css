# Sandbox Component Showcase: Pure-CSS Interactive SVG Coordinate Map and Tooltip Module

## Overview
A high-performance, script-free interactive coordinate blueprint map and data tooltip utility isolated within an examples subdirectory. It handles localized spatial micro-interactions entirely inside the browser's vector tree layer without firing runtime mouse-position event loops.

## 📁 Sandbox Configuration Files
* `demo.html` — Self-contained vector canvas layout hosting server cluster racks linked to adjacent tooltip nodes to evaluate hover velocities.
* `style.css` — Scoped layout modifier asset layer configuring hardware-accelerated transform fields linked back to framework tokens.

## 🚀 Key Layout Mechanics
1. **Adjacent Vector Selector Cascades:** Bypasses client rectangle coordinate lookup calculators by nesting descriptive tooltip blocks (`<g class="alm-map-tooltip">`) directly downstream from target interactive shape layers (`.alm-interactive-node:hover + g`).
2. **Dynamic Vector Sizing Containment:** Employs the `overflow: visible;` parameter rule on the root inline SVG canvas. This ensures absolute tooltip cards extend cleanly outside the core box boundaries without being cropped or layout-shattered.
3. **Sandbox Compliance Safeguard:** Fully satisfies repository security policies by placing every asset line strictly inside `submissions/examples/interactive-svg-map/`.

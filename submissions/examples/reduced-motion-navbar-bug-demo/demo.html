Missing prefers-reduced-motion Guard

The navbar component's background, border, shadow, and link color transitions continue animating even when users request reduced motion.

Bug Demo

Brand
Home About Contact

Hover over the navbar items to see color transitions. Under prefers-reduced-motion: reduce, these should stop — but they don't, because no guard exists in navbar.css.

The Bug

components/navbar.css defines transition on .ease-navbar-glass (lines 29-35) and .ease-navbar-item (lines 65-66) but has no @media (prefers-reduced-motion: reduce) block.

The Fix (in this demo's style.css)

@media (prefers-reduced-motion: reduce) {
  .ease-navbar-glass,
  .ease-navbar-glass .ease-navbar-item {
    transition: none !important;
  }
}

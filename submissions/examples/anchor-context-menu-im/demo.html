Anchor-Positioned Context Menu

Right-click any card below. The menu anchors to the card using the CSS Anchor Positioning API and automatically flips position to stay within the viewport — try near the screen edges. Falls back to JS coordinate positioning in unsupported browsers.

How to Use

css
/* 1. Give the target element an anchor name */
.anchor-target {
  anchor-name: --card-anchor;
}

/* 2. Reference it from the menu, with automatic edge-flipping */
.anchor-menu {
  position: fixed; /* fallback positioning base */
  position-anchor: --card-anchor;
  position-area: bottom span-right;

  /* Try alternate positions if the default overflows the viewport */
  position-try-fallbacks:
    flip-block,
    flip-inline,
    flip-block flip-inline;
}
js — anchor-name assignment + fallback
const menu = document.getElementById('contextMenu');
const supportsAnchor = CSS.supports('anchor-name: --foo');

document.querySelectorAll('.anchor-target').forEach((el, i) => {
  const anchorName = `--card-anchor-${i}`;

  el.addEventListener('contextmenu', (e) => {
    e.preventDefault();

    if (supportsAnchor) {
      // Modern path: assign anchor-name, let CSS position the menu
      el.style.anchorName = anchorName;
      menu.style.positionAnchor = anchorName;
    } else {
      // Fallback: position via getBoundingClientRect()
      const rect = el.getBoundingClientRect();
      menu.style.position = 'fixed';
      menu.style.top = `${rect.bottom + 4}px`;
      menu.style.left = `${rect.left}px`;

      // Flip if menu would overflow viewport edges
      const menuRect = menu.getBoundingClientRect();
      if (rect.left + menuRect.width > window.innerWidth) {
        menu.style.left = `${rect.right - menuRect.width}px`;
      }
      if (rect.bottom + menuRect.height > window.innerHeight) {
        menu.style.top = `${rect.top - menuRect.height - 4}px`;
      }
    }

    menu.classList.add('anchor-menu--open');
  });
});

document.addEventListener('click', () => menu.classList.remove('anchor-menu--open'));
Browser support: CSS Anchor Positioning (anchor-name, position-anchor, position-area, position-try-fallbacks) ships in Chrome/Edge 125+. In unsupported browsers (Firefox, Safari as of this writing), CSS.supports('anchor-name: --foo') returns false and the JS fallback computes position from getBoundingClientRect() with manual edge-flip logic — the menu works identically either way, only the positioning mechanism differs.

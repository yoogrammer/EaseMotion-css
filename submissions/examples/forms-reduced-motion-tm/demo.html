Form Input States — Reduced Motion Guard

Submission proposing that the prefers-reduced-motion: reduce guard in components/forms.css also cover the .ease-input-* state classes. Linked to issue #5576.

Default (no reduced-motion)

Simulating reduced-motion (via media emulation)

Open the browser DevTools, switch to "Emulate CSS prefers-reduced-motion: reduce" to see the proposed guard in action. The state classes will still show the border color, but transitions and box-shadow rings will be neutralized.

Why this matters

Without this guard, users with motion sensitivity still see border-color and box-shadow changes on focus. The proposed change makes the form input state classes just as accessible as the base inputs already are.

.ease-border-primary Fix

Issue #1797 — silent no-op when used standalone

Live comparison

❌ Current (broken)

class="ease-border-primary"

No border visible — border-color alone has no effect when border-style is none (the default).

✅ After fix

class="ease-border-primary"

1px solid primary-color border — self-contained, no companion class needed.

Root cause — core/utilities.css line 238

/* CURRENT — broken */
.ease-border-primary { border-color: var(--ease-color-primary); }

/* FIXED — use full shorthand */
.ease-border-primary { border: 1px solid var(--ease-color-primary); }

Compare with .ease-border (line 236) which correctly uses border: 1px solid …. The fix makes both consistent.

Usage after fix

<!-- Standalone — works without a companion .ease-border class -->
<div class="ease-border-primary ease-rounded ease-padding-4">
  Primary bordered card
</div>

<!-- Composable with other utilities -->
<input class="ease-border-primary ease-rounded" type="text">

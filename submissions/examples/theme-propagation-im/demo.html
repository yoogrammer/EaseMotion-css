How It Works

01

:has() detects selection

.theme-picker:has(input[value="ocean"]:checked) matches the picker container whenever the "Ocean" radio is checked — no JS event listener needed.

02

Custom property cascades down

Each :has() rule sets --live-accent to a different base color. Because it's set on the .theme-picker container, it inherits down into .theme-target-grid and every card.

03

color-mix() derives shades

Borders, backgrounds, badges, progress bars, and buttons all compute their colors with color-mix(in srgb, var(--live-accent) X%, ...) — one variable change updates every derived shade simultaneously.

How to Use

html
<div class="theme-picker">
  <label class="theme-option">
    <input type="radio" name="theme" value="ocean" checked>
    <span class="theme-swatch theme-swatch--ocean"></span> Ocean
  </label>
  <label class="theme-option">
    <input type="radio" name="theme" value="sunset">
    <span class="theme-swatch theme-swatch--sunset"></span> Sunset
  </label>

  <!-- Must be a sibling/descendant the picker can :has()-target -->
  <div class="theme-target-grid">
    <div class="theme-target-card">
      <span class="theme-badge">Active</span>
      <h3>Revenue</h3>
      <div class="theme-metric">$48,210</div>
      <button class="theme-btn">View Details</button>
    </div>
    <!-- more cards... -->
  </div>
</div>
css
/* Each theme option sets --live-accent on the whole picker
   (which contains the target grid) when its radio is checked */
.theme-picker:has(input[value="ocean"]:checked) {
  --live-accent: #0ea5e9;
}

.theme-picker:has(input[value="sunset"]:checked) {
  --live-accent: #f97316;
}

.theme-picker:has(input[value="forest"]:checked) {
  --live-accent: #22c55e;
}

.theme-picker:has(input[value="grape"]:checked) {
  --live-accent: #a855f7;
}

/* Every derived color comes from color-mix() against --live-accent */
.theme-target-card {
  border: 1px solid color-mix(in srgb, var(--live-accent) 30%, transparent);
  background: color-mix(in srgb, var(--live-accent) 6%, var(--ease-color-surface));
}

.theme-badge {
  background: color-mix(in srgb, var(--live-accent) 18%, transparent);
  color: color-mix(in srgb, var(--live-accent) 80%, white);
}

.theme-bar-fill {
  background: linear-gradient(90deg,
    var(--live-accent),
    color-mix(in srgb, var(--live-accent) 50%, white));
}

.theme-btn {
  background: var(--live-accent);
  color: color-mix(in srgb, var(--live-accent) 10%, black);
}
Why this is hard without :has(): Normally, "select A to change B's colors" requires a JS event listener that toggles a class or sets a CSS variable on an ancestor. :has() lets a parent's own styling react to a descendant's state directly in CSS — and because custom properties inherit, that one reactive variable cascades into every color-mix() derivation below it. Browser support: :has() ships in all major browsers (Chrome/Edge 105+, Firefox 121+, Safari 15.4+); color-mix() in Chrome/Edge 111+, Firefox 113+, Safari 16.2+.

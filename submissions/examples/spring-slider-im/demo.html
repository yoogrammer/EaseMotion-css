Physics-Based Spring Slider

Drag any slider below — the thumb and fill settle into position with a spring overshoot, driven by @property custom property interpolation. No JS animation loop, no requestAnimationFrame.

Bouncy — More Overshoot

Stiff — Minimal Overshoot

How It Works

01

@property registration

--spring-offset is registered as an animatable <number>, telling the browser it can interpolate this custom property the same way it interpolates opacity or transform.

02

Overshoot cubic-bezier

A cubic-bezier(0.34, 1.56, 0.64, 1) curve has a control point above 1.0, causing the interpolated value to briefly exceed its target — the spring "boing".

03

Tiny JS sync

A small script reads the native <input type="range"> value on input and writes it to --spring-offset via style.setProperty() — the CSS transition engine handles all motion from there.

How to Use

css
@property --spring-offset {
  syntax: '<number>';
  inherits: false;
  initial-value: 0;
}

.spring-slider-thumb {
  transition: --spring-offset 0.6s cubic-bezier(0.34, 1.56, 0.64, 1);
  left: calc(var(--spring-offset) * 1%);
}

.spring-slider-fill {
  transition: --spring-offset 0.6s cubic-bezier(0.34, 1.56, 0.64, 1);
  width: calc(var(--spring-offset) * 1%);
}

/* Variants tune the spring's stiffness */
.spring-slider--bouncy .spring-slider-thumb,
.spring-slider--bouncy .spring-slider-fill {
  transition-timing-function: cubic-bezier(0.18, 2.2, 0.32, 1);
}

.spring-slider--stiff .spring-slider-thumb,
.spring-slider--stiff .spring-slider-fill {
  transition-timing-function: cubic-bezier(0.34, 1.1, 0.64, 1);
}
js — minimal sync
const input = document.querySelector('.spring-slider-input');
const thumb = document.querySelector('.spring-slider-thumb');
const fill  = document.querySelector('.spring-slider-fill');

input.addEventListener('input', () => {
  const val = input.value;
  thumb.style.setProperty('--spring-offset', val);
  fill.style.setProperty('--spring-offset', val);
});
Browser support: @property is supported in all major browsers (Chrome/Edge 85+, Firefox 128+, Safari 16.4+). Where unsupported, the custom property still works as a static value — the slider functions normally but the spring transition falls back to a standard linear/eased transition on left/width directly (also declared as a fallback, see style.css).

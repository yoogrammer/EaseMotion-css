Missing prefers-reduced-motion Guard

The tabs component's underline slide and label color transitions continue animating even when users request reduced motion. The fix below adds a @media (prefers-reduced-motion: reduce) block.

The Bug

components/tabs.css lines 23 and 60 define transition on .ease-tab-label and .ease-tab-underline but the file has no @media (prefers-reduced-motion: reduce) block. Every other component (cards, buttons, sidebar, footer, etc.) has this guard.

The Fix (applied in this demo's style.css)

@media (prefers-reduced-motion: reduce) {
  .ease-tab-label,
  .ease-tab-underline {
    transition: none !important;
  }
}

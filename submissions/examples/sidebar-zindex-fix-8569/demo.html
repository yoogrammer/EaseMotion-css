Sidebar z-index Fix

Issue #8569 — Uses --ease-z-overlay CSS variable instead of hardcoded z-index: 100

What Changed

Before After
z-index: 100; z-index: var(--ease-z-overlay, 100);
(mobile) z-index: 100; /* hardcoded */ (mobile) z-index: var(--ease-z-overlay, 100); /* variable */

Benefits

  • Consistent with navbar.css which already uses var(--ease-z-overlay, 100)
  • Users can customize overlay z-index globally via --ease-z-overlay
  • Overlay backdrop uses calc(var(--ease-z-overlay, 100) - 1) to stay behind sidebar
  • Toggle button uses calc(var(--ease-z-overlay, 100) + 1) to stay above sidebar

Test It

Resize browser to ≤768px width, then click ☰ to open the sidebar. The sidebar uses the --ease-z-overlay variable for its z-index stack.

aspect-ratio Refactor

By entirely eradicating the notorious "padding-bottom hack" and aggressively adopting the native CSS aspect-ratio property, we completely eliminate unnecessary absolute positioning and heavily reduce DOM bloat across all framework media components.

Legacy Padding Hack

Requires an outer wrapper with padding-bottom: 56.25% and a completely separate absolute-positioned inner child. Creates massive DOM bloat.

16:9 (Bloated DOM)

Modern aspect-ratio

Requires exactly one single element. The browser natively calculates the precise 16:9 ratio instantly. Incredibly clean and performant.

EaseMotion CSS Submission #6013

ease-hover-dim & ease-hover-bright

Two missing hover utilities the cheatsheet already documents (#5716) — brightness-based hover feedback, composable with existing transform/glow hover classes.

Hover Each Card

ease-hover-dim

Dims to 70% brightness on hover

ease-hover-bright

Brightens to 130% on hover

Composed With Existing Hover Classes

grow + bright

Scales up AND brightens together

lift-shadow + dim

Lifts with shadow AND dims together

Common Use Case — Image Galleries

🖼
Hover to dim (common gallery pattern)
🖼
Hover to brighten

How to Use

html
<!-- Basic usage -->
<div class="ease-hover-dim">Dims on hover</div>
<div class="ease-hover-bright">Brightens on hover</div>

<!-- Composable with existing hover classes -->
<div class="ease-hover-grow ease-hover-bright">Grows AND brightens</div>
<div class="ease-hover-lift-shadow ease-hover-dim">Lifts AND dims</div>

<!-- Common use case: image galleries -->
<img class="ease-hover-dim" src="photo.jpg" alt="...">
css
.ease-hover-dim {
  transition: filter var(--ease-speed-medium) var(--ease-ease);
}
.ease-hover-dim:hover {
  filter: brightness(0.7);
}

.ease-hover-bright {
  transition: filter var(--ease-speed-medium) var(--ease-ease);
}
.ease-hover-bright:hover {
  filter: brightness(1.3);
}
Composability: like ease-hover-glow, both classes only transition the filter property, so they stack cleanly with transform-based hover classes (ease-hover-grow, ease-hover-lift-shadow, ease-hover-shrink) without any conflicting transitions.

ease-hover-glow Compose Workaround

The framework .ease-hover-glow sets box-shadow: none on hover, which overrides composed shadow classes like .ease-shadow-lg. This example shows the workaround:

/* Workaround: scope box-shadow:none to only glow-related shadows */
.ease-hover-glow {
  transition: filter var(--ease-speed-medium) var(--ease-ease);
}
.ease-hover-glow:hover {
  filter: drop-shadow(0 0 8px rgba(108,99,255,0.45)) drop-shadow(0 0 20px rgba(108,99,255,0.30));
  box-shadow: none; /* removes framework glow only */
}

✅ Workaround Active

See style.css. The permanent fix in core/animations.css should scope box-shadow: none more carefully.

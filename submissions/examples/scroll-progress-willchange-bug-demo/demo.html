will-change Not Reset Under Reduced Motion

components/scroll-progress.css sets will-change: transform but the reduced-motion block never resets it — wasting GPU compositor layers when no animation runs.

Bug Demo

The progress bar at the top uses will-change: transform. Under prefers-reduced-motion: reduce, the animation stops but will-change remains — the browser keeps a compositor layer allocated.

Enable prefers-reduced-motion: reduce in your browser and inspect the progress bar's computed styles. will-change should be auto but stays transform.

The Bug

Line 52: will-change: transform is set on .ease-scroll-progress.

Lines 103-108: The reduced-motion block stops the animation and sets transform: scaleX(1) but never resets will-change: auto.

The Fix (in this demo's style.css)

@media (prefers-reduced-motion: reduce) {
  .ease-scroll-progress {
    animation: none;
    transform: scaleX(1);
    will-change: auto;
  }
}

Interactive FAQ Accordion Grid

A keyboard-accessible, zero-dependency FAQ grid using modern CSS Grid height transitions and rotating indicator icons.

It transitions the grid-template-rows property from 0fr to 1fr. Because the inner content wrapper is configured with overflow: hidden, the browser smoothly transitions the height to fit the dynamic child text without using any Javascript height calculations.

Yes. The trigger uses a semantic HTML <button> element, making it natively focusable via keyboard Tab keys and toggleable via Space/Enter. It also implements aria-expanded and aria-controls attributes that update dynamically to notify screen readers of status changes.

While checkbox hacks work for pure CSS layouts, they violate semantic accessibility rules. Checkboxes announce themselves as "checkbox state checked/unchecked" to screen readers, which is confusing in a navigation context. Semantic buttons correctly identify as buttons and announce expanded status.

Yes, standard media query overrides are integrated. When a user has prefers-reduced-motion: reduce active in their OS or browser settings, the grid opening transitions and the icon rotation animations are completely bypassed, showing the content instantly without motion.

● Why did the test pass falsely?

Because expect(css).toContain('font-size: 0') searches the entire compiled CSS file as a single string. It matches font-size: 0 inside decimals like font-size: 0.875rem.

Buggy Code tests/smoke.test.js

Asserts a rule that doesn't actually exist on the class, passing by mistake. 

it('should hide text...', () => {
  expect(css).toContain('.ease-btn-loading');
  expect(css).toContain('font-size: 0'); // Falsely matches 0.875rem!
});
Fixed Code tests/smoke.test.js

Asserts the correct styling declaration implemented by the loading state. 

it('should hide text...', () => {
  expect(css).toContain('.ease-btn-loading');
  expect(css).toContain('color: transparent'); // Exact & correct match!
});

Submitted as a test suite bug fix for EaseMotion CSS.

⚡ Repo Hygiene & Contributor Flow

Best practices for contributing to EaseMotion CSS cleanly and consistently.

Contributor Checklist

✓
Add files only inside submissions/examples/ Never edit core/, components/, workflows, or config files.
✓
Include all 3 required files demo.html · style.css · README.md — all three are mandatory.
✓
One feature per PR Keep PRs focused. Don't bundle multiple features together.
✓
Max 2 active issues at a time Finish or unassign one before claiming a new issue.
✓
Use a unique suffix in your folder name e.g. ease-hover-pari, ease-card-ak — avoids naming conflicts.
✗
Never self-merge your PR Only the maintainer (SAPTARSHI-coder) merges pull requests.

Contribution Flow

1
Fork & Clone Fork the repo on GitHub then clone your fork locally.
2
Create a branch git checkout -b feat/your-feature-name
3
Add your submission Create submissions/examples/your-feature-name/ with 3 files.
4
Commit & Push git add · git commit -m "feat: description #issue" · git push
5
Open a Pull Request Fill out the PR template, reference the issue, and wait for review.

Quick Git Commands

# Fork, clone and setup
git clone https://github.com/YOUR-USERNAME/EaseMotion-css
cd EaseMotion-css

# Create feature branch
git checkout -b feat/your-feature-name

# Stage and commit
git add submissions/examples/your-feature/
git commit -m "feat: add your feature #issue-number"

# Push to your fork
git push origin feat/your-feature-name

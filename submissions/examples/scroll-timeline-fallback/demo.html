On unpatched frameworks inside Safari or Mozilla Firefox, dropping native scroll properties triggers a silent failure. Because those engines fail to compile the timeline, the progress bar stays frozen at zero size.

This sandbox model overrides that gap. The system runs a feature capability detection sequence upon page load. If the browser lacks scroll-driven animation components, it mounts a passive scroll event interceptor.

This handler computes the exact vertical displacement offset metrics and binds the percentage fraction directly to a standard custom CSS property variable: --scroll-percent.

By offloading calculation bounds back to reactive style definitions, the fallback tracking track matches native frame updates perfectly across any modern alternative browser engine.

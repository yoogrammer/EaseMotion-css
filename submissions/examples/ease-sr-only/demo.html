ease-sr-only Utility

Visually hide content while keeping it accessible to screen readers

What is ease-sr-only?

The ease-sr-only class visually hides an element from the screen but keeps it fully readable by screen readers and assistive technologies. This is a standard accessibility pattern used by Bootstrap, Tailwind CSS, and every major CSS framework — but it was missing from EaseMotion CSS.

♿
Without this utility, developers are forced to write the same verbose inline styles repeatedly. ease-sr-only makes accessibility a first-class, discoverable feature of the framework.

Live Examples

Icon-only button with hidden label
<button>
  ✕
  <span class="ease-sr-only">Close dialog</span>
</button>

Visually shows only the icon. Screen readers announce "Close dialog".

<a href="#main" 
   class="ease-sr-only ease-sr-only-focusable">
  Skip to main content
</a>

Hidden visually but appears on keyboard focus — essential for keyboard navigation.

Search form with hidden label
<label for="search" class="ease-sr-only">
  Search the site
</label>
<input id="search" type="search" />

Label is invisible but screen readers still announce "Search the site" when the input is focused.

ease-not-sr-only — reveal hidden content
Now you can see me! 👋 
<!-- Hidden -->
<div class="ease-sr-only">Hidden text</div>

<!-- Revealed -->
<div class="ease-not-sr-only">Visible text</div>

ease-not-sr-only reverses the effect — useful for toggling visibility dynamically.

The Classes — CSS

css — add to core/utilities.css
/* ── Screen Reader Only ─────────────────────────────────── */

/* Visually hide but keep accessible to screen readers */
.ease-sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border-width: 0;
}

/* Reveal on focus — for skip links */
.ease-sr-only-focusable:focus,
.ease-sr-only-focusable:focus-within {
  position: static;
  width: auto;
  height: auto;
  padding: 0.5rem 1rem;
  margin: 0;
  overflow: visible;
  clip: auto;
  white-space: normal;
  background: var(--ease-color-primary);
  color: #ffffff;
  border-radius: var(--ease-radius-md);
  z-index: var(--ease-z-toast);
}

/* Reverse sr-only — make visible again */
.ease-not-sr-only {
  position: static;
  width: auto;
  height: auto;
  padding: 0;
  margin: 0;
  overflow: visible;
  clip: auto;
  white-space: normal;
}

/* High Contrast Mode support */
@media (forced-colors: active) {
  .ease-sr-only {
    border: 0;
  }
}

When to use each class

Class Use case
ease-sr-only Hide labels, descriptions, or text visually but keep for screen readers
ease-sr-only-focusable Skip navigation links — hidden until keyboard focus, then revealed
ease-not-sr-only Undo sr-only — useful when toggling visibility via JavaScript

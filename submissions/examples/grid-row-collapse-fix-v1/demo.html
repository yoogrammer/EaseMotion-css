Grid Row Collapse & Overlay Fix

Using fluid fractional grid rows inside auto-filling frameworks can cause WebKit (iOS/Safari) to collapse row heights to 0. This utility locks tracks to safe limits.

⚠️ Collapsed WebKit Grid

Uses repeat(2, minmax(0, 1fr)). In iOS Safari, dynamic rendering blurs can collapse heights and overlay cards on top of each other.

Grid Item Alpha

This is dynamic card content. In bugged layouts, it overlaps with beta due to dynamic height calculation glitches.

Grid Item Beta

This item is forced into the second row, but overlaps the first item when parent height collapses to 0.

grid-template-rows: repeat(2, minmax(0, 1fr))

✨ Fixed Guard Grid

Uses the ease-grid-row-guard utility. Safely measures content bounds and locks dynamic rows without layout clipping.

Grid Item Alpha

This is dynamic card content. The row guard preserves natural heights, preventing overlap completely.

Grid Item Beta

This item sits safely in its row, maintaining visual gaps regardless of WebKit viewport constraints.

grid-auto-rows: minmax(min-content, max-content)

Footer & Sidebar Fallback Color Fix

Issue #8567 — Replaces incorrect fallback #111 with the actual neutral-900 value #0f172a

The Problem

Both components/footer.css (line 10) and components/sidebar.css (line 18) use #111 as the fallback for --ease-color-neutral-900, but the actual variable value is #0f172a (dark navy-slate). These are noticeably different colors.

Token Fallback (Before) Actual Value Fallback (After)
--ease-color-neutral-900 #111 #0f172a #0f172a

Affected Files

FileLineBeforeAfter
components/footer.css 10 var(--ease-color-neutral-900, #111) var(--ease-color-neutral-900, #0f172a)
components/sidebar.css 18 var(--ease-color-neutral-900, #111) var(--ease-color-neutral-900, #0f172a)

Why It Matters

If the CSS variable --ease-color-neutral-900 fails to resolve (due to layer ordering, CDN issues, or conditional loading), the footer and sidebar will display as pure near-black (#111) while every other component using the same token defaults to the correct navy-slate (#0f172a). This creates a visual inconsistency.

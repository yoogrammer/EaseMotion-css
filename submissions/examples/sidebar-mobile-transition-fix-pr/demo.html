🐛 Sidebar Mobile Slide-in Fix

Replaces display:none/block with visibility/pointer-events to enable the CSS slide-in transition on mobile.

Interactive Demo (simulated mobile viewport)

EaseApp 📊 Dashboard 👤 Profile ⚙️ Settings 📁 Files

Main Content

Click the button above to open the sidebar. With the fix, it smoothly slides in from the left with a bounce easing. Without the fix, it snaps into view instantly.

Before vs After

❌ Before (broken)
/* Hidden state */
display: none;
transform: translateX(-100%);

/* Open state */
display: block;
transform: translateX(0);

/* display is NOT animatable!
   The transition never fires. */
✅ After (fixed)
/* Hidden state */
visibility: hidden;
pointer-events: none;
transform: translateX(-100%);

/* Open state */
visibility: visible;
pointer-events: auto;
transform: translateX(0);

/* transform IS animatable!
   Smooth slide-in works. */
How to test:
1. Click "☰ Open Sidebar" in the mobile frame above — the sidebar smoothly slides in from the left.
2. Click the button again to close — the sidebar smoothly slides back out.
3. The bounce easing cubic-bezier(0.34, 1.56, 0.64, 1) is the same one already defined in sidebar.css line 82, which was never working due to the display:none issue.
4. Without the fix: the sidebar snaps instantly because display:none → display:block is not an animatable transition.

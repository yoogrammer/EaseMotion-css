ease-ping Scale Fix

Replaces hardcoded scale(2) with var(--ease-ping-scale, 2)

BEFORE — Buggy scale(2) hardcoded — overlaps content on large elements

All three elements use the buggy ease-ping-buggy with hardcoded scale(2). Works fine on the tiny dot but destroys layout on the badge and button — the ping ring grows huge and covers surrounding content.

8px dot — looks okay
48px badge — ring too large!
99+
Button — ring huge, overlaps!

AFTER — Fixed CSS variable controls scale per element

Same three elements using the fixed ease-ping. Each one sets --ease-ping-scale to a value that looks right for its size — no keyframe override needed.

8px dot — scale: 2 (default)
48px badge — scale: 1.5
99+
Button — scale: 1.2

The Fix — CSS

css
/* BEFORE (buggy) — in core/animations.css */
@keyframes ease-kf-ping {
  75%, 100% {
    transform: scale(2);   /* ← hardcoded, no override possible */
    opacity: 0;
  }
}

/* AFTER (fixed) — replace in core/animations.css */
@keyframes ease-kf-ping {
  75%, 100% {
    transform: scale(var(--ease-ping-scale, 2)); /* ← CSS variable */
    opacity: 0;
  }
}

/* Usage — no keyframe override needed */
.notification-badge {
  --ease-ping-scale: 1.5;   /* smaller ring for large elements */
}

.tiny-dot {
  --ease-ping-scale: 2.5;   /* bigger ring for tiny elements */
}

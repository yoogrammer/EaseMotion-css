EaseMotion CSS — Submission

Spacing Scale Step 5 (spacing-5-mab)

Adds ease-padding-5 and ease-margin-5 — the only missing utilities in the spacing scale. The token --ease-space-5 (1.25rem / 20px) already exists; this submission exposes it as utility classes.

Problem
Spacing Scale Coverage Gap

Every token in the spacing scale except step 5 has a corresponding utility class. The table below shows the complete picture.

Token Value Padding Class Margin Class
--ease-space-1 0.25rem ✅ ease-padding-1 ✅ ease-margin-1
--ease-space-2 0.5rem ✅ ease-padding-2 ✅ ease-margin-2
--ease-space-3 0.75rem ✅ ease-padding-3 ✅ ease-margin-3
--ease-space-4 1rem ✅ ease-padding-4 ✅ ease-margin-4
--ease-space-5 1.25rem ✦ ease-padding-5 (added) ✦ ease-margin-5 (added)
--ease-space-6 1.5rem ✅ ease-padding-6 ✅ ease-margin-6
--ease-space-8 2rem ✅ ease-padding-8 ✅ ease-margin-8

Usage
HTML Snippet

Drop-in usage after integration by the maintainer.

<!-- All sides -->
<div class="ease-padding-5">Padded content</div>
<div class="ease-margin-5">Spaced content</div>

<!-- Composing both -->
<section class="ease-padding-5 ease-margin-5">
  Consistent spacing at every scale step.
</section>

<!-- Axis variants -->
<div class="ease-padding-x-5">Horizontal padding only</div>
<div class="ease-padding-y-5">Vertical padding only</div>

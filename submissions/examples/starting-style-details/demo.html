FAQ

What is @starting-style?

@starting-style is a CSS at-rule that defines the starting state for elements before they are first rendered or when they transition from display: none to visible. It enables smooth entry animations for elements like details, dialogs, and popovers.

What is transition-behavior: allow-discrete?

The allow-discrete keyword enables transitions on discrete properties like display and content-visibility. Combined with @starting-style, it lets you animate elements entering or leaving the DOM.

Why use discrete property transitions?

Discrete properties (display, visibility) normally can't be transitioned. With transition-behavior: allow-discrete and @starting-style, you can create smooth open/close animations for disclosure widgets, modals, and more.

Browser support?

@starting-style and transition-behavior: allow-discrete are supported in Chrome 117+, Edge 117+, and Safari 17.4+. Firefox has not yet implemented support.

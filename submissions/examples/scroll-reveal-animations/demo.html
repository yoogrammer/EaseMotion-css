01 — Core Classes

Six Reveal Variants

↑ .em-reveal-up

Fades in and slides upward as the element enters the viewport. Great for cards, headings, and hero content.

.em-reveal-up

↓ .em-reveal-down

Fades in and slides downward. Works well for navigation dropdowns or elements anchored to the top.

.em-reveal-down

← .em-reveal-left

Slides in from the left side of the viewport.

.em-reveal-left

→ .em-reveal-right

Slides in from the right side of the viewport.

.em-reveal-right

⊕ .em-reveal-zoom

Zooms in from a smaller scale. Ideal for images, badges, and call-to-action elements that deserve emphasis.

.em-reveal-zoom

◎ .em-reveal-fade

Simple opacity fade with no transform. The most subtle option — great for text blocks and backgrounds.

.em-reveal-fade
02 — Modifiers

Speed Control

Fast Reveal

Uses .em-reveal--fast — animation completes earlier as you scroll in.

.em-reveal-up.em-reveal--fast

Default Speed

Default range: entry 0% entry 40%

.em-reveal-up

Slow Reveal

Uses .em-reveal--slow — animation plays over a longer scroll distance, more cinematic feel.

.em-reveal-up.em-reveal--slow

04 — Mixed Usage

Real-World Patterns

"EaseMotion's scroll reveals align with the animation-first philosophy — native CSS, no JavaScript bundle cost, graceful degradation built in."

✦ No JS

Zero runtime JavaScript. Works entirely in CSS using modern scroll-driven animation APIs.

♿ Accessible

Respects prefers-reduced-motion — all animations are disabled for users who prefer it.

🔧 Fallback

Gracefully degrades to a simple fade in browsers without animation-timeline support via @supports.

⚡ Lightweight

Pure CSS — no dependencies, no build step. Drop in style.css and add classes.

ease-btn-loading Fix

Fixes layout shift caused by spinner being appended after button text

BEFORE — Buggy Spinner appended after text — button gets wider

Notice the loading button is wider than the normal button. In a button group this pushes adjacent buttons out of place.

Normal button
Loading button (buggy — wider!)
Button group — loading breaks layout

AFTER — Fixed Spinner overlays text — button stays same width

The fixed version hides text with color: transparent, positions the spinner absolutely in the center, and locks button width with min-width so layout never shifts.

Normal button
Loading button (fixed — same width!)
Button group — layout stays intact

The Fix — CSS

css
/* BEFORE (buggy) */
.ease-btn-loading::after {
  content: '';
  display: inline-block;       /* appended AFTER text */
  width: 0.85em;
  height: 0.85em;
  border: 2px solid currentColor;
  border-top-color: transparent;
  border-radius: 50%;
  animation: ease-kf-rotate 0.7s linear infinite;
  margin-left: var(--ease-space-2);  /* pushes button wider */
}

/* AFTER (fixed) */
.ease-btn-loading {
  pointer-events: none;
  position: relative;
  color: transparent !important;      /* hide text */
  min-width: var(--ease-btn-min-w, 0); /* lock width */
}

.ease-btn-loading::after {
  content: '';
  position: absolute;                 /* overlay, not append */
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  width: 1em;
  height: 1em;
  border: 2px solid rgba(255,255,255,0.4);
  border-top-color: #ffffff;
  border-radius: 50%;
  animation: ease-kf-rotate 0.7s linear infinite;
}

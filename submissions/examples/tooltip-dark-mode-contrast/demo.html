Tooltip Dark Mode Contrast Fix

tooltips.css dark mode uses near-white bg on dark page — inconsistent with all other dark-mode surfaces

Enable dark mode: Chrome DevTools → More Tools → Rendering → set Emulate CSS media feature prefers-color-scheme to dark. Then hover the buttons in each section to see the tooltip color difference.
Broken Current — near-white tooltip box on dark background

In dark mode the tooltip background becomes #f1f5f9 (near-white) with #0f172a (near-black) text. Both tokens — --ease-color-neutral-100 and --ease-color-neutral-900 — have no dark mode override in variables.css, so they stay at their light-mode values. A bright white box on a dark page is jarring and inconsistent with cards, modals, and navbar which all use dark surfaces in dark mode.

Fixed Uses dark surface token — consistent with other components

The fix sets --ease-tooltip-bg to --ease-color-surface (#141e33 in dark mode) and --ease-tooltip-text to --ease-color-text (#e2e8f0 in dark mode). Both tokens are correctly overridden by variables.css. The tooltip now uses the same dark surface style as cards, modals, and the navbar in dark mode.

Live — real .ease-tooltip classes with fix applied

These use the real .ease-tooltip class. style.css overrides the dark mode token values. In dark mode they now match the dark surface appearance of the rest of the framework.

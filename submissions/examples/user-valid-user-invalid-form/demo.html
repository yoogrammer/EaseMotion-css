How :user-valid/:user-invalid differs from :valid/:invalid

  • :valid/:invalid — fire immediately, even on untouched fields
  • :user-valid/:user-invalid — only fire after user interaction (typing, blur, form submission attempt)
  • Empty required fields don't show :user-invalid until the user tabs through or submits
  • Results in less aggressive, more user-friendly validation UX

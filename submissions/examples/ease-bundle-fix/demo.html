🔧 Bundle Sync Fix

Fix for Issue #6937 — Stale easemotion.min.css breaks release validation

❌ The Problem

Running npm run release:check or npm run validate:bundle fails because easemotion.min.css is out of sync with the source CSS files.

❌ Error: easemotion.min.css is stale. Run `npm run build` and commit the regenerated bundle.
> easemotion-css@1.0.0 validate:bundle > node scripts/validate-bundle.mjs Error: easemotion.min.css is stale. Run `npm run build` and commit the regenerated bundle.

✅ The Fix — Step by Step

1 Install dependencies
npm install
2 Regenerate the minified bundle
npm run build
3 Verify both validation scripts pass
npm run validate:bundle npm run release:check
4 Commit the updated bundle
git add easemotion.min.css git commit -m "chore: regenerate minified CSS bundle"
✅ Bundle is up to date — all validation checks pass

⚠️ How to prevent this in future

⚠️ Always run npm run build before committing source CSS changes. Never manually edit easemotion.min.css directly.

Scroll-Linked Progress Ring

Scroll this page — the ring in the top-right fills via conic-gradient driven by animation-timeline: scroll(root). No scroll event listener, no requestAnimationFrame.

How It Works

01

@property registers --ring-pct

--ring-pct is registered as an animatable <number> via @property, so it can be used directly inside a conic-gradient() and animated.

02

animation-timeline: scroll(root)

Instead of a normal time-based animation, animation-timeline: scroll(root) ties the keyframe progress directly to the document's scroll position — 0% scrolled = animation at 0%, 100% scrolled = animation at 100%.

03

conic-gradient renders the ring

The ring's background is conic-gradient(var(--ease-color-primary) calc(var(--ring-pct) * 1%), transparent 0) — as --ring-pct increases from the scroll-linked animation, the colored arc grows around the circle.

Scroll Through This Content

Section 1

This is a long article to demonstrate the scroll-linked ring. As you scroll down through these sections, watch the ring in the top-right corner fill up and the percentage label update — all without a single scroll event listener.

The CSS Scroll-Driven Animations specification introduces animation-timeline, which can be set to scroll() to tie a CSS animation's progress to a scrollable element's scroll offset rather than to wall-clock time.

Section 2

Combined with @property, which allows custom properties to be registered with a specific syntax (here, <number>) and therefore be animatable, we can drive arbitrary visual properties — like the percentage in a conic-gradient() — directly from scroll position.

This is fundamentally different from the older approach of listening to the scroll event and manually computing window.scrollY / (document.body.scrollHeight - window.innerHeight) on every frame. The browser's compositor handles everything, often off the main thread.

Section 3

The percentage label inside the ring is updated via a tiny scroll listener (for text content, since CSS can't yet inject computed numbers into text nodes) — but the ring's visual fill itself is 100% CSS-driven and will continue animating smoothly even if JS is blocked or slow.

This hybrid approach — CSS for the visual progress, minimal JS for the text label — represents a practical, production-ready pattern for scroll progress indicators.

Section 4

Keep scrolling — the ring should now be most of the way full. Notice there's no jank or stutter, even on long pages, because the browser doesn't need to re-run JavaScript on every scroll frame for the visual fill.

animation-timeline: scroll(root) uses the document's root scroller by default. You can also scope it to a specific scrollable container by giving that container a scroll-timeline-name and referencing it with animation-timeline: --my-timeline.

Section 5 — End

You've reached the end. The ring should now read 100%. Scroll back up to watch it unfill in reverse — scroll-driven animations are bidirectional by nature, since they're a direct function of scroll position, not elapsed time.

How to Use

css
@property --ring-pct {
  syntax: '<number>';
  inherits: false;
  initial-value: 0;
}

.scroll-ring {
  width: 56px;
  height: 56px;
  border-radius: 50%;
  background: conic-gradient(
    var(--ease-color-primary) calc(var(--ring-pct) * 1%),
    rgba(255,255,255,0.1) 0
  );

  animation: ring-fill linear;
  animation-timeline: scroll(root);
}

@keyframes ring-fill {
  from { --ring-pct: 0; }
  to   { --ring-pct: 100; }
}
js — label text only
// CSS handles the ring's visual fill entirely.
// This tiny script only updates the text label inside it,
// since CSS can't yet write computed numbers into text content.
const label = document.querySelector('.scroll-ring-label');

function updateLabel() {
  const scrolled = window.scrollY;
  const max = document.documentElement.scrollHeight - window.innerHeight;
  const pct = Math.round((scrolled / max) * 100);
  label.textContent = `${pct}%`;
}

document.addEventListener('scroll', updateLabel, { passive: true });
updateLabel();
Browser support: animation-timeline: scroll() ships in Chrome/Edge 115+ and Firefox 131+ (Safari support pending as of this writing). Where unsupported, @supports not (animation-timeline: scroll()) falls back to the JS-computed percentage driving --ring-pct directly via style.setProperty() on the same scroll listener already used for the label — see style.css for the full fallback block.

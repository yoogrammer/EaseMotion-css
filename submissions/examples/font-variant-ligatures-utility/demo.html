Font-Variant-Ligatures Utility

Controls ligature rendering — the way certain character pairs (fi, fl, ffi) are combined into single glyphs.

Serif font: ligature comparison

.font-variant-ligatures-normal ff fi fl ffi ffl (browser default)
.font-variant-ligatures-none ff fi fl ffi ffl (no ligatures)
.font-variant-ligatures-common ff fi fl ffi ffl (common ligatures only)
.font-variant-ligatures-discretionary ff fi fl ffi ffl (discretionary ligatures)
.font-variant-ligatures-historical ff fi fl ffi ffl (historical ligatures)
.font-variant-ligatures-contextual ff fi fl ffi ffl (contextual ligatures)

Mono font: code vs. prose

.font-variant-ligatures-none (code) function initialize() { return ffmpeg != null; }
-> != and == are visually distinct
.font-variant-ligatures-common (code) function initialize() { return ffmpeg != null; }
-> fi and ff ligatures active

"No-" classes to selectively disable

.font-variant-ligatures-no-common ff fi fl ffi ffl
.font-variant-ligatures-no-discretionary ff fi fl ffi ffl
.font-variant-ligatures-no-historical ff fi fl ffi ffl
.font-variant-ligatures-no-contextual ff fi fl ffi ffl
Tip: Code editors and code blocks typically disable ligatures with .font-variant-ligatures-none to keep characters distinct. Decorative headings and pull quotes often enable .font-variant-ligatures-discretionary for a more refined typographic appearance.

Non-Blocking Font Loading

Fix for core/base.css — Issue #1796

❌ Current (broken) CSS @import — render-blocking

core/base.css currently contains this at line 6: 

@import url('https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap');

A CSS @import inside a stylesheet blocks the browser from rendering anything on the page until the imported file is fully downloaded and parsed. This adds a full network round-trip to the critical rendering path.

Run Lighthouse → Performance on any page using EaseMotion CSS and it will flag this under "Eliminate render-blocking resources".

✅ Fixed approach <link> tags — non-blocking

The maintainer should remove the @import from core/base.css and add these tags to docs/index.html and all example <head> blocks: 

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap" rel="stylesheet">

<link rel="preconnect"> opens the TCP connection to Google's servers immediately — in parallel with all other asset downloads. The font is discovered and fetched as early as possible, never blocking the render tree.

The framework already declares fallbacks in --ease-font-sans: 

'Inter', system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

So removing the @import from core/base.css causes zero functional regression — pages without the <link> tags simply render using the system font.

📖 README update Document the recommended snippet

The README "Getting Started" section should include: 

<!-- Optional: load Inter font (recommended for best appearance) -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap"
      rel="stylesheet">

<!-- EaseMotion CSS -->
<link rel="stylesheet" href="easemotion.css">

This makes font loading an explicit, informed choice — not a hidden side-effect of importing the framework.

💡 How to verify: Open DevTools → Network tab → reload with throttling set to "Slow 3G". With the @import, the waterfall shows a blocking chain. With <link> tags, font and CSS downloads run in parallel.

01

The Browser Compatibility Gap

Modern CSS features like scroll-driven animations (animation-timeline: scroll()) enable extremely performant scroll indicators that run entirely on the compositor thread. By shifting progress calculation away from main-thread JavaScript execution, modern browsers bypass common scroll stutter (jank).

However, browsers lacking support for scroll-driven animations (such as Safari on iOS and older desktop engines) fail to interpret these properties. Without progressive enhancement, the bar remains completely hidden (transform: scaleX(0)) or static, deteriorating the premium user experience.

02

Graceful Degradation Design

This solution utilizes a double-layer strategy to support 100% of modern and legacy viewports:

By default, the progress bar uses standard custom properties: transform: scaleX(var(--scroll-percentage, 0)). In unsupported viewports, a lightweight, passive JavaScript event listener tracks scroll offsets and adjusts this custom property.

How it upgrades: If the browser supports native scroll-driven timelines, the JS execution is completely bypassed using an early return. The stylesheet upgrades the element using an @supports (animation-timeline: scroll()) query, initializing high-performance GPU-bound rendering.
03

Passive Event Listeners

When using JavaScript for scroll tracking, performance is critical. Standard scroll event handlers can block scroll rendering because the browser waits for the event handler to finish before updating layout offsets.

To prevent this, the fallback script registers listeners using { passive: true }. This tells the browser that the scroll handler will not call preventDefault(), allowing the browser to scroll the viewport instantly while processing our layout updates asynchronously.

04

Accessibility Compliance (WCAG 2.1)

EaseMotion CSS is built to respect accessibility settings. Under WCAG 2.1 Success Criterion 2.3.3 (Animation from Interaction), users must have a way to disable non-essential motion.

This feature handles motion reduction seamlessly:

If a user has prefers-reduced-motion: reduce active:
1. The fallback JavaScript listener returns immediately and performs no actions.
2. The CSS media query disables keyframe animations and displays a static indicator, keeping the layout clean and functional.

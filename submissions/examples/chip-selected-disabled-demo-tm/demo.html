Chip Selected and Disabled Demo

Self-contained demo of the chip's :checked and :disabled states. Linked to issue #5507 which proposed smoke-test coverage for the sibling selector chain.

Why this matters

The chip component is implemented entirely in CSS using the hidden-input + sibling-label pattern. The selected and disabled states are produced by two specific sibling-selector chains: input[type="checkbox"]:checked + .ease-chip and input[type="checkbox"]:disabled + .ease-chip. If either is removed, the visual states break silently.

Social Hover translateY Not Reset

The footer's reduced-motion guard removes the transition on social links but doesn't reset the :hover translateY(-2px). The icon still jumps — an instant movement violates WCAG 2.1 SC 2.3.3.

Bug Demo

Enable prefers-reduced-motion: reduce and hover over the social icons. The transition is gone but the icon still jumps up by 2px because transform: translateY(-2px) on :hover is not cleared.

The Bug

components/footer.css line 81: .ease-footer-social a:hover { transform: translateY(-2px); }

Lines 168-173: The reduced-motion block sets transition: none !important on the links but doesn't reset transform: none on the hover state.

The Fix (in this demo's style.css)

@media (prefers-reduced-motion: reduce) {
  .ease-footer-links a,
  .ease-footer-social a {
    transition: none !important;
    transform: none !important;
  }
  .ease-footer-social a:hover {
    transform: none !important;
  }
}

Architecture & Cascade

Explicit CSS Layer Cascade Order

Define a predictable cascade hierarchy inside EaseMotion CSS by declaring the layer order explicitly: @layer easemotion-base, easemotion-components, easemotion-utilities;. This prevents source-order dependency bugs across bundlers.

Interactive Cascade Simulation

Normally in CSS, if a utility class is defined before a component class in the stylesheet, the component's styles will win due to source order—even if the developer wants the utility to override the component. CSS Cascade Layers solve this by decoupling style specificity from source order.

Cascade Resolution Mode

/* Legacy Source-Order Cascade */

/* 1. Utilities (imported first) */
.ease-mt-4 {
  margin-top: 1rem;
}

/* 2. Components (imported second) */
.ease-btn {
  padding: 0.75rem 1.25rem;
  margin-top: 0.5rem; /* ← Overrides the utility due to source order! */
}

Architectural Advantages

Predictable Hierarchy

Explicit layer order ensures that utility layers (`easemotion-utilities`) always override components (`easemotion-components`) regardless of where or in what order files are imported or bundled.

Bundler Independence

Build utilities, Sass/Less preprocessors, webpack, or Vite can concatenate CSS in differing source orders. An explicit layer declaration makes order variation safe.

Extendability

Allows developers to easily inject their custom styles either before, between, or after EaseMotion layers without having to resort to fragile `!important` flags.

Integration Guide

To integrate this architecture into the main EaseMotion CSS bundle, declare the layer order at the very beginning of core/variables.css or index.css:

index.css
/* Explicit layer hierarchy declaration at the top of the bundle */
@layer easemotion-base, easemotion-components, easemotion-utilities;

/* Import framework source files scoped to their respective layers */
@import "core/variables.css" layer(easemotion-base);
@import "components/buttons.css" layer(easemotion-components);
@import "components/cards.css" layer(easemotion-components);
@import "utilities/spacing.css" layer(easemotion-utilities);

ease-min-h-0 / ease-min-w-0

Flex & grid overflow fix utilities — Issue #1804

💡 Why this matters: .ease-truncate already exists in core/utilities.css but silently fails inside a flex row. These two utilities are the missing pair that make it work correctly.

Exact addition for core/utilities.css

/* OVERFLOW section, around line 166 */

.ease-overflow-hidden  { overflow: hidden; }
.ease-overflow-auto    { overflow: auto; }
.ease-overflow-scroll  { overflow: scroll; }
.ease-overflow-x-auto  { overflow-x: auto; }
.ease-overflow-y-auto  { overflow-y: auto; }

/* Flex / grid overflow fix — overrides implicit min-height/width: auto */
.ease-min-h-0 { min-height: 0; }
.ease-min-w-0 { min-width:  0; }

Pairing with the existing ease-truncate utility

<!-- ease-truncate alone silently fails in a flex row -->
<div class="ease-flex ease-gap-4">
  <img src="avatar.jpg" style="width:40px; flex-shrink:0;">
  <div class="ease-flex-1">                <!-- ❌ text overflows -->
  <div class="ease-flex-1 ease-min-w-0">  <!-- ✅ truncation works -->
    <p class="ease-truncate">Long username that truncates correctly</p>
  </div>
</div>

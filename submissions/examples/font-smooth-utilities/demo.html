Font Smooth Utility Classes

Override -webkit-font-smoothing and -moz-osx-font-smoothing per element. Best seen on macOS with a retina display. The difference is most visible on dark backgrounds and at medium font weights.

💡 Note: Font smoothing differences are most visible on macOS Safari and Chrome on HiDPI (Retina) displays. On Windows and Linux the properties have no effect — text renders the same regardless.

Light background comparison

.ease-antialiased

The quick brown fox jumps over the lazy dog. Smooth antialiasing makes text appear lighter and crisper on HiDPI screens.

.ease-subpixel

The quick brown fox jumps over the lazy dog. Subpixel rendering makes text appear slightly heavier — better for small body text on LCD displays.

Dark background comparison (most visible difference)

.ease-antialiased

Antialiased text on dark backgrounds looks clean and sharp — the recommended setting for light-on-dark UI text.

.ease-subpixel

Subpixel rendering on dark backgrounds can look heavy or blurry on some displays. Generally avoid for dark UIs.

Reset to browser default

.ease-font-smooth-auto

Browser default smoothing — useful to reset a section that inherits the global antialiased base from core/base.css.

.ease-font-smooth-none

No smoothing — rarely used but available for special cases like pixel-art or bitmap-style text effects.

Common use cases

  • Dark mode UI — add .ease-antialiased to text containers
  • Hero headings on dark hero sections
  • Reset a component that overrides the global base setting
  • Small body copy on light backgrounds — try .ease-subpixel

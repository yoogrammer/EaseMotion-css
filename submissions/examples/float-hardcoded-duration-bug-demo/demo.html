Bug: hardcoded 3s duration

The float animation uses 3s instead of a CSS variable. The keyframe also lacks the ease-kf- prefix convention.

Floating element

Source (from core/animations.css line 683)

.ease-float {
  animation: ease-float 3s ease-in-out infinite;  /* hardcoded 3s */
}

Fix

.ease-float {
  animation: ease-float var(--ease-float-speed, 3s) ease-in-out infinite;
}

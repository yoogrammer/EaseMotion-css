What you're seeing:

When reduced motion is disabled (default), all three bounce classes play their full keyframe animations — the box bounces vertically, scales in, or shrinks out.

When reduced motion is enabled (click the button above), the @media (prefers-reduced-motion: reduce) guard collapses every animation to 0.01ms with a single iteration. The boxes snap instantly to their final visual state with no visible motion.

On a real device with prefers-reduced-motion: reduce set in the OS accessibility settings, the browser applies this automatically — no toggle needed. This demo simulates that behavior via a class toggle so you can preview the effect without changing your system settings.

The guard shown here is missing from the current easemotion/bounce.css. Every other animation file (fade.css, slide.css, rotate.css, zoom.css, hover.css, misc.css, timing.css) already has it. This is a WCAG 2.1 SC 2.3.3 violation.

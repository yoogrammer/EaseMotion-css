Troubleshooting

Common Issues & Solutions

Step-by-step fixes for the most frequent EaseMotion CSS setup problems.

⚠️
Animations Not Triggering
Classes applied but nothing moves
  1. CSS not loaded — open DevTools → Network, filter by .css, confirm easemotion.css returns HTTP 200.
  2. Class name typo — check for em- prefix; class names are case-sensitive.
  3. Missing fill mode — add em-fill-both so the end state persists.
  4. Element hidden — if the element starts as display:none, the animation fires but is invisible. Use opacity:0 + em-fade-in instead.
  5. Reduced-motion preference — the library respects prefers-reduced-motion. Test in a standard browser profile.
💡
Quick check: paste document.querySelector('.em-fade-in') in the console. If null, the class isn't on the element.
🚫
CSS Import Problems
Styles fail to load or apply
  1. Wrong import path — use a path relative to your HTML file, not the project root.
  2. CDN typo — copy the CDN link directly from the README; a single wrong character breaks it silently.
  3. MIME type blocked — some servers return text/plain for .css files. Fix your server's MIME config.
  4. CORS error (local dev) — open files with http:// via a dev server (npx serve .), not file://.
  5. Specificity clash — another stylesheet may override animation rules. Increase specificity or use !important sparingly.
/* HTML — link tag */ <link rel="stylesheet" href="./css/easemotion.min.css"> /* JS/TS — bundler import */ import 'easemotion-css/dist/easemotion.min.css';
🔧
Build Tool Configuration
Vite / Webpack / Parcel not bundling CSS
  1. Vite — import in main.js; Vite handles CSS auto. Ensure css.modules is not set to true globally.
  2. Webpack 5 — requires css-loader + style-loader (or MiniCssExtractPlugin). Check your webpack.config.js rules array.
  3. PostCSS / PurgeCSS — safelist all em-* classes so they aren't purged from production builds.
  4. Parcel — no config needed; import CSS directly in JS. Clear .parcel-cache if stale.
  5. Next.js — import in _app.js (App Router: layout.tsx). Do not import inside component files.
/* PurgeCSS safelist (postcss.config.js) */ module.exports = { plugins: { '@fullhuman/postcss-purgecss': { safelist: [/^em-/] } } }
🌐
Browser Compatibility
Animations work in some browsers but not others
  1. Safari / WebKit — 3D transforms need -webkit- prefix on older versions. Use Autoprefixer in your PostCSS pipeline.
  2. iOS Safari — position:fixed + transform can conflict; test on a real device, not just simulator.
  3. Firefox — fully supported. If an animation glitches, check will-change conflicts.
  4. IE 11 — not supported. CSS custom properties (var()) and modern keyframe syntax are not available.
  5. Old Edge (EdgeHTML) — upgrade to Chromium-based Edge; no workaround for the legacy engine.
ℹ️
Add Autoprefixer to your PostCSS config to auto-generate -webkit- and -moz- prefixes for maximum coverage.
📜
Scroll-Triggered Animations
Animations fire immediately instead of on scroll
  1. No JS integration — EaseMotion CSS classes trigger on load, not on scroll. You must add/remove classes via IntersectionObserver.
  2. Observer threshold too high — if threshold: 1.0, the element must be 100% visible. Use 0.2 for earlier triggering.
  3. Elements below the fold pre-animated — add visibility:hidden or opacity:0 to off-screen elements and apply the animation class only in the observer callback.
/* Minimal scroll-trigger setup */ const observer = new IntersectionObserver( (entries) => entries.forEach(e => { if (e.isIntersecting) e.target.classList.add('em-slide-up'); }), { threshold: 0.2 } ); document.querySelectorAll('[data-animate]') .forEach(el => observer.observe(el));
🐢
Janky / Dropped Frames
Animations stutter or run below 60 fps
  1. Animating layout properties — never animate width, height, top, left, or margin. Use only transform and opacity.
  2. Too many simultaneous animations — stagger animations with em-delay-1 / em-delay-2 / em-delay-3.
  3. Missing will-change — add will-change: transform, opacity to heavily animated elements to promote them to their own compositor layer.
  4. Overdraw — open Chrome DevTools → Rendering → enable "Paint flashing". Green flashes on every frame indicate expensive repaints.
  5. Low-end devices — check prefers-reduced-motion and offer a no-animation fallback.

⚡ Quick-Diagnosis Checklist

# Check How to Verify
1 CSS file loaded with status 200 DevTools → Network tab → filter by .css
2 Correct class name with em- prefix DevTools → Elements → inspect class attribute
3 No prefers-reduced-motion active DevTools → Rendering → Emulate: no preference
4 Element is visible (display ≠ none) DevTools → Computed → check display
5 No conflicting animation rule elsewhere DevTools → Styles → look for strikethrough rules
6 PurgeCSS not removing em-* classes Search final CSS bundle for em-fade-in
7 Build tool processing CSS correctly Check bundler output / console for CSS errors
Compatibility

Browser & Framework Support

EaseMotion CSS targets modern browsers. For legacy support, add Autoprefixer.

Browser / Environment Status Minimum Version Notes
Chrome / Edge (Chromium) ✅ Full 60+ All features supported
Firefox ✅ Full 55+ All features supported
Safari / iOS Safari ⚠️ Partial 12+ Add Autoprefixer; test 3D on real device
Samsung Internet ✅ Full 8+ Chromium-based; no issues
Internet Explorer 11 ❌ None — No CSS custom properties support
React / Next.js ✅ Full Any Import in _app.js / layout.tsx
Vue / Nuxt ✅ Full Any Import in main.js or nuxt.config CSS array
Angular ✅ Full Any Add path to angular.json styles array
Svelte / SvelteKit ✅ Full Any Import in +layout.svelte
Vite ✅ Full 2+ Auto-handles CSS; no extra config needed
Webpack 5 ✅ Full 5+ Requires css-loader + style-loader
Parcel 2 ✅ Full 2+ Zero config
⚠️
Using PurgeCSS or Tailwind's JIT purge? Add /^em-/ to your safelist regex to prevent EaseMotion classes from being removed in production builds.

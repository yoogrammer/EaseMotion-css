Fix #2776: Manual CSS concatenation lacks newlines

tests/smoke.test.js joins 16 CSS files with the + operator and no separators. If any file is missing a trailing newline (e.g. ends with /* end of component */ and no \n), the next file's first rule merges onto the same line and can be silently commented out.

Illustration of the bug

// File A ends without newline: .ease-fade-in { opacity: 1; } /* end of component */.ease-card { ... } // ↑ .ease-card rule is now INSIDE the comment — silently broken!

Fixed result with array + join("\n")

// File A ends without newline: .ease-fade-in { opacity: 1; } /* end of component */ .ease-card { ... } // ↑ newline guarantees .ease-card starts a fresh line

Proposed tests/smoke.test.js diff

- css = variables + base + animations + utilities + buttons + cards
-     + chip + footer + masonry + navbar + scrollProgress + sidebar
-     + tabs + badges + loaders + tooltips + modals;

+ css = [
+   variables, base, animations, utilities,
+   buttons, cards, chip, footer, masonry, navbar,
+   scrollProgress, sidebar, tabs, badges, loaders,
+   tooltips, modals
+ ].join("\n");

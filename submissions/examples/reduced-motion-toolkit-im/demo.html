The Reduced-Motion Layer

css — global safety net (core/base.css)
/* Matches the issue's suggested global reset, scoped to avoid
   breaking scroll-snap / scroll-behavior utilities that some
   pages rely on even with reduced motion (those are handled
   separately and intentionally NOT included here). */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-delay: -1ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    transition-delay: 0s !important;
  }

  html {
    scroll-behavior: auto;
  }
}
css — opt-in .ease-motion-safe wrapper
/* For authors who want an EXPLICIT, scoped guarantee that a
   section's animations/transitions are fully disabled under
   reduced motion — independent of whether every individual
   component remembers its own @media block. Wrap any container: */
@media (prefers-reduced-motion: reduce) {
  .ease-motion-safe,
  .ease-motion-safe *,
  .ease-motion-safe *::before,
  .ease-motion-safe *::after {
    animation: none !important;
    transition: none !important;
    transform: none !important;
    scroll-behavior: auto !important;
  }
}
Why both layers? The global * rule (0.01ms duration) is a safety net that catches any animation/transition framework-wide, including ones a component author forgot to guard individually — matching the issue's suggested solution exactly. The opt-in .ease-motion-safe wrapper goes further: it fully removes transform too (not just speeds up the transition to it), for sections where even an instant transform-jump would be jarring (e.g. parallax, tilt, magnetic-hover effects where the resting `transform` itself is part of the motion).

How to Use

html
<!-- The global safety net applies automatically — no class needed -->
<div class="ease-pulse">...</div>

<!-- For sections where even instant transform-jumps are unwanted,
     wrap with .ease-motion-safe for a stronger guarantee -->
<section class="ease-motion-safe">
  <div class="magnetic-hover-card">...</div>
</section>

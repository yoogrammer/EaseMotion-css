Fix for #14085: Tabs Nav Border Layer Cascade

When an unlayered CSS reset (like Tailwind's Preflight) applies * { border-width: 0; }, it overrides the layered border-bottom: 2px on .ease-tabs-nav because unlayered styles have higher priority than layered ones.

After (Fixed using box-shadow)

By replacing border-bottom with box-shadow: inset 0 -2px 0 0, the visual line is retained because box-shadow is not typically reset globally.

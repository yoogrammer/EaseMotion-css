Double Animation Fix Demo

Demonstrates the conflict when two animation classes are applied to the same element, and the correct fix.

❌ Broken — Two classes (zoom-in overrides slide-up)

Using both ease-slide-up and ease-zoom-in — only zoom-in plays

Broken: ease-slide-up + ease-zoom-in

✅ Fixed — Combined class

Using ease-slide-up-zoom-in — both animations play together

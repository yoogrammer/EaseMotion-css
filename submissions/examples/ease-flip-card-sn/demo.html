HTML Structure

The flip card follows a predictable three-layer structure.

.flip-card

Outer container

Establishes the 3D perspective context. Sets the card dimensions.

.flip-card-inner

Middle wrapper

Handles the rotation animation with transform-style: preserve-3d.

.flip-card-front

Visible side

The default face visible before hover. No rotation applied.

.flip-card-back

Hidden side

Rotated 180deg by default. Revealed when the card is hovered.


      

    


    
    

      
CSS Custom Properties

      
Customize the flip behavior by overriding these CSS variables on the .flip-card element.

      

        

          
--flip-duration

          Default: 0.6s
          
Controls the speed of the flip animation. Use any valid CSS time value like 0.3s or 1s.

        

        

          
--flip-axis

          Default: Y
          
Sets the rotation axis. Use X for vertical flip or any other valid rotation axis.

        

        

          
--flip-width

          Default: 300px
          
Sets the card width. Accepts any valid CSS length or percentage.

        

        

          
--flip-height

          Default: 200px
          
Sets the card height. Accepts any valid CSS length or percentage.

        

      


      

        

      

    


    
    

      
Use Cases

      
Flip cards are widely used across modern web interfaces. Here are some common scenarios.

      

        

          
👥

          
Team Members

          
Show a photo and role on the front, bio and social links on the back. Perfect for about pages.

        

        

          
🛍️

          
Product Showcases

          
Display product image and name on the front, price and description on the back.

        

        

          
🏆

          
Portfolio Items

          
Project thumbnail on the front, details and live link on the back after flip.

        

        

          
🎮

          
Interactive Galleries

          
Create engaging gallery experiences where each card reveals additional information.

        

      

    


    
    

      
Accessibility

      
The component respects prefers-reduced-motion — the flip animation is completely disabled for users who prefer reduced motion, ensuring a comfortable experience for everyone.

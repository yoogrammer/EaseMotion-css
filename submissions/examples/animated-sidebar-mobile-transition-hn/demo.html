Mobile Transition Fix

Sidebar Animation Correction

Issue Fixed: Previously, on viewports under 768px, the sidebar used a binary display: none style to hide. When toggled, the browser switched display: none to display: block in the same frame as transform: translateX(0), skipping the transition entirely and causing a jarring visual snap. Reverting also snapped closed immediately.

We fixed this by replacing display logic with CSS visibility: hidden and pointer-events: none, preserving slide-in and slide-out animation transitions.

Testing instructions:

1. Open this page and **resize your browser window to under 768px** (or view in mobile responsive mode).
2. Click the toggle button. The sidebar menu will slide in smoothly from the left side.
3. Click the toggle button again. The sidebar menu will animate and slide back out smoothly instead of disappearing instantly.

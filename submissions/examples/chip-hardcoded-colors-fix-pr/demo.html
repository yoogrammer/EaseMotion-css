🐛 Chip Hardcoded Colors Fix

Replaces hardcoded #ffffff text and static rgba() shadows with design tokens in the selected chip state.

Theme Override Demo

Notice how the selected chips now respect the customized text color (--ease-selection-color) and shadow (--ease-color-primary-alpha) for their respective themes. Without the fix, the yellow chip's text would be unreadable white, and its shadow would be blue.

Yellow Theme (needs dark text)

Teal Theme (needs light text)

How to test:
1. Observe the "Yellow Theme" chip. Because yellow is a light color, we override --ease-selection-color to dark slate. The selected chip correctly uses dark text and a yellow shadow.
2. Observe the "Teal Theme" chip. It uses white text and a teal shadow.
3. Without the fix: The selected chip text is strictly hardcoded to #ffffff (white), making it totally unreadable against a yellow background. The shadow is also strictly hardcoded to rgba(99, 102, 241, 0.3) (indigo), creating a mismatched blue shadow on yellow/teal chips.

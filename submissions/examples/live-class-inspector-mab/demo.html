How the Inspector Works

The inspector is a single floating panel injected once into the DOM. It reads from HTML data-* attributes — no CSS parsing or getComputedStyle required.

01

Annotate your demo box

Add data-ease-class and data-ease-rule to any .anim-box element. The data attributes are the single source of truth.

02

Hover or Tab-focus the box

The JS listener fires on mouseenter and focus. It reads the data attributes, updates the inspector content, positions the panel near the target element, and fades it in.

03

Inspector renders class + rule

The class badge and tokenised animation rule are updated synchronously. No layout recalculation — the panel simply repositions based on getBoundingClientRect().

04

One-click copy

Copy Class writes the bare class string to the clipboard. Copy HTML writes a ready-to-paste <div class="ease-…"></div> snippet. Both buttons flash a green "Copied ✓" confirmation.

05

Inspector hides on leave or blur

Fires on mouseleave / blur with a small 150ms debounce so clicking the copy buttons doesn't accidentally close the panel.

04

HTML Markup

Add the data-ease-class and data-ease-rule attributes to any demo element.

demo.html 
<!-- Shared inspector panel (inject once into body) -->
<div
  id="demo-inspector"
  class="demo-inspector"
  role="tooltip"
  aria-live="polite"
></div>

<!-- Animation demo boxes -->
<div
  class="anim-box ease-slide-up"
  tabindex="0"
  data-ease-class="ease-slide-up"
  data-ease-rule="animation: ease-kf-slide-up 300ms cubic-bezier(0.4, 0, 0.2, 1) both"
>
  Slide Up
</div>

06

Accessibility

♿ The inspector uses role="tooltip" and aria-live="polite" so screen readers announce the class and rule when the panel appears. All demo boxes have tabindex="0" for keyboard navigation and descriptive aria-label attributes. Copy buttons are native <button> elements with explicit aria-label text. The @media (prefers-reduced-motion: reduce) block disables all animations while keeping the inspector fully functional.

:is() vs :where()

Understanding specificity differences between modern CSS pseudo-class functions — no JavaScript required.

:is() — Takes highest specificity

:is(.special, div) wins over a plain element selector because it takes the highest specificity from its arguments (class = 0,1,0 beats element = 0,0,1).

I'm an <span> with .special — also matched

:where() — Specificity is always 0

:where(.special, div) never beats a plain element selector because its specificity is always zero.

I'm not highlighted — :where() can't override the element selector

Neither am I

Direct Comparison

Same structure — :is() overrides the base style, :where() does not.

:is(.special, p) — overrides base (specificity 0,1,0)

:where(.special, p) — base overrides this (specificity 0,0,0)

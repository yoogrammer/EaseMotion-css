MatchMedia Feature Detection

Calling window.matchMedia without a feature detection guard can throw in older browsers or testing environments.

Before: No guard

Direct call to window.matchMedia without checking if it exists.

// UNSAFE: no feature detection
const mq = window.matchMedia(
  '(prefers-reduced-motion: reduce)'
);

if (mq.matches) {
  disableAnimations();
}
❌ Throws ReferenceError if matchMedia is undefined

Risks

  • Breaks in older browsers (IE, older Safari)
  • Throws in SSR/JSDOM environments
  • No graceful degradation path

After: With guard

Feature-detects window.matchMedia before calling it, with a safe fallback.

// SAFE: feature detection guard
const hasMatchMedia =
  typeof window !== 'undefined' &&
  typeof window.matchMedia === 'function';

const mq = hasMatchMedia
  ? window.matchMedia(
      '(prefers-reduced-motion: reduce)'
    )
  : { matches: false };

if (mq.matches) {
  disableAnimations();
}
✔ Gracefully falls back to { matches: false }

Benefits

  • Works in all browsers
  • Safe in SSR/testing environments
  • Graceful fallback behavior

Recommended pattern

Always guard window.matchMedia and other browser APIs with a feature detection check:

function getMatchMedia(query) {
  if (typeof window !== 'undefined'
      && typeof window.matchMedia === 'function') {
    return window.matchMedia(query);
  }
  return { matches: false, addListener() {}, removeListener() {} };
}

// Usage
const mq = getMatchMedia('(prefers-reduced-motion: reduce)');
if (mq.matches) { /* ... */ }

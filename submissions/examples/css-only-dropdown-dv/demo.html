Why CSS-Only?

⌨️ Full Keyboard Accessibility

By adding tabindex="0" to the dropdown wrapper and using the CSS :focus-within pseudo-class, the menu naturally reveals when keyboard focus enters, allowing screen readers and keyboard users to navigate link options instantly.

🚀 Zero JS Overhead

No listeners, zero execution loops, and no layout shift issues. Browsers compute visibility transitions natively at 60+ FPS, aligning with EaseMotion CSS's ultra-performant design philosophy.

✨ Built-in Animation Integration

Uses a smooth 3D translate-and-scale physics entry combined with EaseMotion's existing ease-kf-slide-down keyframes for premium transitions.

.ease-dropdown {
  position: relative;
  display: inline-block;
}

.ease-dropdown-menu {
  position: absolute;
  top: calc(100% + var(--ease-space-2, 0.5rem));
  left: 0;
  z-index: 1000;
  min-width: 12rem;
  background-color: var(--ease-color-surface, #ffffff);
  border: 1px solid var(--ease-color-neutral-200, #e2e8f0);
  border-radius: var(--ease-radius-md, 0.5rem);
  box-shadow: var(--ease-shadow-lg);
  opacity: 0;
  visibility: hidden;
  transform: translateY(-0.5rem) scale(0.95);
  transition: opacity 150ms var(--ease-ease),
              transform 150ms var(--ease-ease),
              visibility 150ms var(--ease-ease);
  pointer-events: none;
}

/* Hover & Focus state activations */
.ease-dropdown:hover .ease-dropdown-menu,
.ease-dropdown:focus-within .ease-dropdown-menu {
  opacity: 1;
  visibility: visible;
  transform: translateY(0) scale(1);
  pointer-events: auto;
}

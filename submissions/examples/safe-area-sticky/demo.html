EaseMotion CSS — Safe-Area Sticky

Sticky Header & Footer

Scroll down. The header stays below the device status bar / notch, and the footer stays above the home indicator bar — using env(safe-area-inset-*).

↓ Scroll down ↓

  • Why safe-area matters

    On iPhones with a notch and Android devices with gesture navigation, a plain position: sticky; top: 0 header overlaps the system status bar. env(safe-area-inset-top) shifts the header down by exactly the right amount.

  • How it works

    The viewport-fit=cover meta tag tells the browser to extend the viewport under the notch. Then padding-top: env(safe-area-inset-top) on the sticky header adds just enough padding to push content below the notch.

  • Desktop preview

    On a desktop browser the safe-area insets are all zero, so the header and footer behave exactly like a regular sticky element. Open this demo on a real iOS device or enable a notched device in DevTools to see the difference.

  • Zero JavaScript

    The entire effect is two CSS rules — one for the top inset, one for the bottom inset. No resize observers, no JavaScript checks, no library overhead.

  • Graceful fallback

    Browsers that do not support env() ignore the padding line and fall back to the flat 0 value declared first. The sticky behaviour still works; the inset padding is simply not applied.

  • Composable

    Apply em-sticky-safe-top to any element you want to stick at the top of the viewport — nav, toolbar, announcement banner. Apply em-sticky-safe-bottom to anything that should hug the bottom — action bars, tab bars, cookie banners.

Badges Missing prefers-reduced-motion Guard

The badge pulse animation (.ease-badge-pulse::after) uses animation: ease-kf-ping ... infinite with no @media (prefers-reduced-motion: reduce) block.

Bug Demo

Default Danger Success

Under prefers-reduced-motion: reduce, the pulsing ring should stop — but it continues because no guard exists in badges.css.

The Bug

components/badges.css line 61 defines animation: ease-kf-ping 1.5s ease-in-out infinite on .ease-badge-pulse::after but has no @media (prefers-reduced-motion: reduce) block.

The Fix (in this demo's style.css)

@media (prefers-reduced-motion: reduce) {
  .ease-badge-pulse::after {
    animation: none !important;
  }
}

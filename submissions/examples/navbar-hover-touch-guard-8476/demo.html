Navbar Hover Touch Guard Fix

Issue #8476 — .ease-navbar-item:hover causes sticky hover on touch devices

How to reproduce the bug Open this page in Chrome DevTools → toggle Device Toolbar → select any mobile preset (e.g. iPhone 14). Tap each nav item in the two navbars below and observe the difference in behaviour.

Broken Without (hover: hover) guard — current behaviour in navbar.css

Logo
Home Docs Components GitHub

On touch: tap any link — the color changes to var(--ease-color-text) and stays stuck until you tap elsewhere. This creates a false "active" state. The :hover rule has no pointer guard so the browser's synthetic tap-hover fires and never resets.

Fixed With @media (hover: hover) and (pointer: fine) guard

Logo
Home Docs Components GitHub

On touch: tap any link — no color change occurs. The hover rule is wrapped in @media (hover: hover) and (pointer: fine), which only matches real pointer devices (mouse, trackpad). Touch devices are excluded entirely. On desktop with a mouse, hover still works correctly.

The Fix

One change to components/navbar.css — wrap the existing rule in the pointer guard: 

/* Before — fires on touch tap */
.ease-navbar-glass .ease-navbar-item:hover {
  color: var(--ease-color-text, #1e293b);
}

/* After — only fires on hover-capable pointer devices */
@media (hover: hover) and (pointer: fine) {
  .ease-navbar-glass .ease-navbar-item:hover {
    color: var(--ease-color-text, #1e293b);
  }
}

ease-card-image Aspect Ratio Fix

Replaces hardcoded height: 200px with aspect-ratio: 16/9

BEFORE — Buggy Hardcoded height: 200px — distorts on wide cards

All three cards below have the same height: 200px image. On wide containers the image stretches horizontally and destroys the aspect ratio — especially visible on the full-width card.

Narrow Card

Looks okay here because card is narrow.

Medium Card

Starting to stretch a little.

Wide Card

Image is very distorted here!

Full Width Card — Image Distorted

The image is severely stretched. Fixed 200px height on a 1200px wide card completely destroys the aspect ratio.

AFTER — Fixed aspect-ratio: 16/9 — scales naturally at any width

Same three cards, same images. Now using aspect-ratio: 16/9 with height: auto — images always look correct at any width.

Narrow Card

Perfect aspect ratio maintained.

Medium Card

Scales perfectly with container.

Wide Card

No distortion at all!

Full Width Card — Image Perfect

The image scales naturally to any container width while maintaining the correct 16:9 aspect ratio.

CSS variable allows custom aspect ratios

The fix uses --ease-card-img-ratio so developers can override per card without specificity fights.

Default — 16:9

Override — 4:3

Override — 1:1 Square

The Fix — CSS

css
/* BEFORE (buggy) */
.ease-card-image .ease-card-img {
  width: 100%;
  height: 200px;        /* ← hardcoded, distorts on wide cards */
  object-fit: cover;
  display: block;
  border-radius: var(--ease-radius-lg) var(--ease-radius-lg) 0 0;
}

/* AFTER (fixed) */
.ease-card-image .ease-card-img {
  width: 100%;
  height: auto;                                    /* ← no fixed height */
  aspect-ratio: var(--ease-card-img-ratio, 16/9); /* ← scales naturally */
  object-fit: cover;
  display: block;
  border-radius: var(--ease-radius-lg) var(--ease-radius-lg) 0 0;
}

/* Custom ratio override — no specificity fight needed */
.my-card {
  --ease-card-img-ratio: 4/3;
}

Print-Safe Animation Utilities

Open your browser's print preview (Ctrl/Cmd + P) to see the difference. On screen, this page has fade-in animations, a scroll-revealed card, and a continuous spinner. In print, animations stop, hidden content becomes visible, and sections break cleanly across pages.

Animated Elements (frozen in print)

Pulsing badge

On screen: spinner rotates, badge pulses, button bounces on hover. In print: all three render as static — .ease-print-safe * sets animation: none !important; transition: none !important;.

Scroll-Revealed Content

This content normally waits for scroll

On screen, this card starts at opacity: 0; transform: translateY(40px) and animates in only when scrolled into view. Without print handling, it would print as a blank gap. With .ease-print-show, it's forced to opacity: 1; transform: none; visibility: visible in print — guaranteed visible regardless of scroll state.

New Page in Print

This section has .ease-print-break-before — in the printed/PDF output, it starts on a fresh page via break-before: page. On screen, this has zero visual effect.

Avoid Breaking Mid-Card

Keep me together

Without .ease-print-avoid-break, a long card near a page boundary might get split — header on one page, body on the next. break-inside: avoid keeps this card on a single page whenever it fits.

How to Use

html
<!-- Apply to <body> or any container to enable print-safe animation freezing -->
<body class="ease-print-safe">

  <!-- Hide screen-only chrome (nav, decorative animations) from print -->
  <header class="ease-print-hide">...</header>

  <!-- Force scroll-revealed content visible in print -->
  <div class="ease-card ease-scroll-reveal ease-print-show">
    Always printable, regardless of scroll state
  </div>

  <!-- Start a new printed page before this section -->
  <section class="ease-print-break-before">...</section>

  <!-- Keep this card on one printed page -->
  <div class="ease-card ease-print-avoid-break">...</div>

</body>
css
@media print {
  /* Freeze all animations and transitions */
  .ease-print-safe *,
  .ease-print-safe *::before,
  .ease-print-safe *::after {
    animation: none !important;
    transition: none !important;
  }

  /* Force hidden/animated-in content to be visible */
  .ease-print-show {
    opacity: 1 !important;
    transform: none !important;
    visibility: visible !important;
    clip-path: none !important;
    max-height: none !important;
  }

  /* Page-break controls */
  .ease-print-break-before { break-before: page; }
  .ease-print-break-after  { break-after: page; }
  .ease-print-avoid-break  { break-inside: avoid; }

  /* Hide screen-only chrome */
  .ease-print-hide { display: none !important; }

  /* Sensible print defaults */
  .ease-print-safe {
    background: #fff !important;
    color: #000 !important;
  }
}
Why this matters: Animated dashboards, pricing pages, and scroll-revealed landing pages often print as broken or blank — content that waits for a scroll trigger never appears in print preview, mid-animation states can get frozen on a half-transformed frame, and long cards split awkwardly across page boundaries. This utility set handles all three cases with four small, composable classes.

CSS-only Tabs

Hidden radio buttons drive the active panel — no JavaScript

HTML Structure

Tabs use hidden <input type="radio"> elements with the same name attribute. Each label's for attribute targets a radio. The :checked pseudo-class reveals the corresponding panel via the general sibling combinator ~.

<input type="radio" name="t" id="tab1" checked>
<label for="tab1">Tab 1</label>
<div class="panel">Content 1</div>

CSS Styling

Radios and non-active panels are hidden. Active panel gets opacity: 1; pointer-events: auto. A sliding underline indicator tracks the active tab using translateX() driven by :has() selectors.

.panel { opacity: 0; pointer-events: none; }
#tab-html:checked ~ .panels #panel-html { opacity: 1; }

JavaScript (Not Needed)

All tab state is managed by the CSS cascade. The radio group ensures only one tab is active at a time. No event listeners, no hash manipulation, no aria sync needed for basic functionality.

For accessibility, add role="tablist", role="tab", and role="tabpanel" attributes alongside the CSS.

API Reference

  • :checked — target the selected radio
  • ~ general sibling combinator
  • opacity + pointer-events for show/hide
  • transition for smooth fading
  • :has() for indicator positioning

Getting Started

EaseMotion CSS is a zero-dependency, animation-first CSS framework. It provides human-readable class names for building polished interfaces with expressive motion. No build step, no complex setup.

The framework ships as a single CSS file that you can link directly via CDN or install from npm. All animations respect prefers-reduced-motion by default.

Installation

The fastest way to get started is through the jsDelivr CDN. Simply add a link tag to your HTML head section. No account or API key required.

For npm users, run npm install easemotion-css and import the stylesheet in your project. The package includes both minified and unminified versions.

You can also use granular imports to load only the animation categories you need. This reduces the total CSS payload for production applications.

Basic Usage

Add class names directly to your HTML elements. The class name describes the behavior in plain English. For example, ease-fade-in applies a fade-in animation, and ease-slide-up slides the element up from below.

Stack multiple classes to compose behaviors. Add ease-delay-200 for a 200ms delay, or ease-duration-slow for a slower animation. Classes are designed to be composable without specificity conflicts.

Customization

Override any CSS custom property to theme the framework. Change colors, spacing, border radius, and animation speeds by redefining variables in your stylesheet.

The framework uses CSS cascade layers, so your custom styles always win without needing !important. Simply define your overrides in the easemotion.components layer.

Accessibility

Every animation in EaseMotion CSS respects the prefers-reduced-motion media query. When a user has reduced motion enabled in their operating system, all animations are automatically disabled or simplified.

Interactive components include proper ARIA attributes, focus management, and keyboard navigation support. The framework follows WCAG 2.1 AA guidelines.

Browser Support

EaseMotion CSS works in all modern evergreen browsers including Chrome 49+, Firefox 31+, Safari 9.1+, and Edge 15+. Internet Explorer is not supported.

The framework uses CSS custom properties, CSS animations, Flexbox, and CSS Grid. For older Safari versions, -webkit-backdrop-filter is automatically applied as a fallback.

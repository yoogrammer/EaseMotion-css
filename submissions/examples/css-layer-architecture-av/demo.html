@layer Architecture Refactor

By wrapping the entire EaseMotion framework inside an @layer easemotion; block, downstream enterprise developers can finally override our components using simple, weak selectors without ever needing to use the toxic !important hack.

Specificity War Defeated

The framework tries to paint this button purple using a highly specific selector:
.ease-demo-btn[data-variant="primary"]:hover.

The user's custom CSS successfully overrides it to black using a terribly weak selector:
.ease-demo-btn.

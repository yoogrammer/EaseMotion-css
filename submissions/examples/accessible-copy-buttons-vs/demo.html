Accessible Copy Buttons

Copy buttons in documentation need descriptive aria-label attributes so screen-reader users know exactly what will be copied — not just "Copy".

The Problem

Generic labels like "Copy" are announced identically by screen readers no matter which button is focused. A user tabbing through several copy buttons hears "Copy, Copy, Copy" with no context about what each button copies.

❌ Inaccessible — all buttons sound the same

--ease-speed-fast
ease-btn-primary

The Fix

Adding a descriptive aria-label gives screen-reader users the same context that sighted users infer from nearby text.

✅ Accessible — each button has a unique label

--ease-speed-fast
--ease-speed-normal
ease-btn-primary
ease-fade-in

Accessibility Features Used

  • Descriptive aria-label — overrides visible button text for screen readers so each button has a unique, meaningful announcement.
  • Keyboard accessible — all buttons are native <button> elements, reachable and activatable with Tab and Enter / Space.
  • Visible focus ring — :focus-visible shows a clear outline for keyboard users without affecting mouse interactions.
  • Live region announcement — a visually-hidden aria-live="polite" region announces "Copied!" to screen readers after a successful copy.
  • Reduced motion — the feedback transition respects prefers-reduced-motion: reduce.

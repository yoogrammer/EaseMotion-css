🐛 Tabs Reduced Motion Fix

Adds missing prefers-reduced-motion: reduce support to the tabs component.

Interactive Tabs

Profile Content

This is the content for the first tab. When motion is enabled, the underline slides and this content fades in smoothly.

Settings Content

This is the content for the second tab. If you enable reduced motion, the sliding underline and the fade-in animation are both disabled instantly.

Billing Content

This is the content for the third tab. It demonstrates how accessibility features like 'prefers-reduced-motion' should be universally supported across components.

How to test:
1. Click between the tabs above — the underline should slide and the content should fade in smoothly.
2. Enable prefers-reduced-motion: reduce in your OS:
    • Windows: Settings → Accessibility → Visual effects → turn off "Animation effects"
    • macOS: System Settings → Accessibility → Display → "Reduce motion"
    • Chrome DevTools: Ctrl+Shift+P → "Emulate CSS prefers-reduced-motion: reduce"
3. Click between the tabs again — the tab switch should now be instant with no animations or transitions.
4. Without the fix (comment out style.css), the animations continue running even with reduced motion enabled.

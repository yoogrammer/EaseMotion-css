♿ Accessibility Guide

How to use EaseMotion CSS accessibly — for everyone

🩷 prefers-reduced-motion

Respect user preferences for reduced motion. Some users experience dizziness or discomfort from animations.

/* Add to your CSS */
@media (prefers-reduced-motion: reduce) {
    *,
    *::before,
    *::after {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
        scroll-behavior: auto !important;
    }
}

🔇 ease-sr-only (Screen Reader Only)

Hide content visually while keeping it accessible to screen readers.

.ease-sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
}

🎨 Color Contrast Guidance

Ensure text meets WCAG 2.1 AA standards (minimum 4.5:1 ratio for normal text).

✅ Good Contrast

#1e293b on #ffffff (15.3:1)
#667eea on #ffffff (8.5:1)

❌ Poor Contrast

Poor visibility (1.2:1)
Hard to read (1.5:1) 
<!-- Always ensure sufficient contrast -->
<button class="ease-btn ease-btn-primary">Good Contrast</button>
<button style="background: #ccc; color: #eee;">❌ Bad Contrast</button>

🏷️ ARIA Attributes

Use ARIA attributes alongside CSS classes for better accessibility.

<!-- Button with ARIA label -->
<button class="ease-btn" aria-label="Close menu">✕</button>

<!-- Toggle with ARIA states -->
<button class="ease-btn" aria-expanded="false" aria-controls="panel">
    Toggle Panel
</button>

<!-- Loading state -->
<div class="ease-loader" aria-live="polite" aria-label="Loading content"></div>

🎯 Focus Indicators

Never remove focus outlines without providing a visible alternative.

/* Good focus indicator */
*:focus-visible {
    outline: 2px solid #667eea;
    outline-offset: 2px;
    border-radius: 4px;
}

/* ❌ Never do this */
/* :focus { outline: none; } */

📄 Semantic HTML

Use semantic HTML elements for better accessibility out of the box.

<!-- ✅ Good -->
<nav class="ease-navbar">
    <ul>
        <li><a href="/">Home</a></li>
    </ul>
</nav>

<!-- ❌ Bad (use CSS to style semantics, don't strip them) -->
<div class="ease-navbar">
    <div class="nav-item">Home</div>
</div>

✅ Accessibility Checklist

📋 WCAG 2.1 AA Guidelines Reference

  • 1.1.1 - Non-text Content: Provide text alternatives
  • 1.4.3 - Contrast (Minimum): 4.5:1 ratio
  • 2.3.3 - Animation from Interactions: Allow reduced motion
  • 4.1.2 - Name, Role, Value: Use ARIA attributes

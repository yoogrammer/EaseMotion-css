Bug Fix #4452: Loading Button Spinner

The current components/buttons.css sets color: transparent on .ease-btn-loading, which makes the ::after spinner border invisible because it uses currentColor. This demo shows the broken vs fixed behavior side by side.

Broken (current core)

Spinner animates but is invisible

Fixed (this submission)

Visible spinner on all variants

Root cause

.ease-btn-loading { color: transparent !important; }
.ease-btn-loading::after { border: 2px solid currentColor; }
/* currentColor === transparent → invisible spinner */

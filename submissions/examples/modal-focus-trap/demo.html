Modal Focus Trap

Fixes Issue #4220. When a modal opens, focus is locked inside it. Tab and Shift+Tab cycle only through the modal's interactive elements. Focus returns to the trigger on close. Background content is hidden from screen readers via aria-hidden.

How to test the focus trap:
1. Open any modal below using the button or link trigger.
2. Press Tab repeatedly — focus must stay inside the modal.
3. Press Shift+Tab to cycle backwards — must wrap to last element.
4. Press Esc or click the × button — focus must return to the trigger.
5. Try a screen reader (NVDA / VoiceOver) — background content must be silent.

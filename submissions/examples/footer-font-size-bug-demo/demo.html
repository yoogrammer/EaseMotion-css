Hardcoded font-size in Footer

components/footer.css uses hardcoded font-size values (0.875rem, 0.9rem, 0.8rem) instead of the --ease-text-* token system. Users customizing typography via tokens will find the footer unresponsive.

Click the buttons to override --ease-text-sm and --ease-text-xs — the footer base font and bottom bar font won't respond because they use hardcoded values.

The Bug

Line 14: font-size: 0.875rem — should use var(--ease-text-sm, 0.875rem)

Line 30: font-size: 0.9rem — no matching token

Line 151: font-size: 0.8rem — no matching token

The Fix (in this demo's style.css)

.ease-footer {
  font-size: var(--ease-text-sm, 0.875rem);
}
.ease-footer-col-title {
  font-size: var(--ease-text-base, 1rem);
}
.ease-footer-bottom {
  font-size: var(--ease-text-xs, 0.75rem);
}

Excessive -v1 Suffixes #4583

Repeated -v1 suffixes in submission folder names bloat directory listings, create git noise, and make PR reviews harder.

core_changes-v1-v1-*
16
-v1-v1 doubled folders
1
-v1-v1-v1 tripled

Naming Convention (for CONTRIBUTING.md)

✅ Use kebab-case with a single unique identifier
❌ No cumulative version suffixes — append -v1 only if a -v2 actually exists
✅ Core changes: core_changes-issue-{number}
✅ Examples: ease-{feature}-{initials}
❌ Single-version folders should have no suffix

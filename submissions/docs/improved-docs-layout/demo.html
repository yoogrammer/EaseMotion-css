Introduction

EaseMotion CSS

A human-readable, animation-first CSS library. Write UI with simple, English-like class names — no memorizing utilities, no complex configuration. Just readable HTML that looks great and moves beautifully.

💡
EaseMotion CSS bridges the gap between raw vanilla CSS and utility-heavy frameworks. You get structure and power — with none of the cognitive overhead.

Layout Improvements — Issue #1539

This submission addresses the following issues from the original docs page:

IssueFix Applied
Content left-aligned with large whitespace on right Max-width container (1400px) centered with margin: 0 auto
No visual distinction between heading levels Clear size/weight/letter-spacing scale for h1, h2, h3
Flat, plain sidebar Gradient active state, animated hover with left border
No visual engagement on landing Animated particle canvas + floating gradient orbs in hero
Dividers look like plain lines Gradient dividers fading to transparent
Tables have no border radius Rounded tables with overflow hidden

Animations

Add animation classes directly to any HTML element.

ClassEffect
ease-fade-in Fades from 0 → 1 opacity
ease-slide-up Slides up while fading in
ease-zoom-in Scales from 85% with elastic ease
ease-bounce Repeating vertical bounce
ease-pulse Gentle scale pulse

Buttons

Ready-to-use button components with variants and sizes.

html 
<button class="ease-btn ease-btn-primary">Primary</button>
<button class="ease-btn ease-btn-primary ease-btn-sm">Small</button>
<button class="ease-btn ease-btn-primary ease-btn-pill">Pill</button>

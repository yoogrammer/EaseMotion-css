Footer Keyframe Namespace Collision Demo

This demo reproduces the keyframe namespace collision described in Issue #27881. It shows how generic keyframe names like float and heartbeat can conflict with application CSS, and how using prefixed names such as ease-kf-footer-float and ease-kf-footer-heartbeat avoids the conflict.

Application Animation

The application defines its own generic @keyframes float animation.

Current Footer (Collision)

The footer also uses the generic @keyframes float, which shares the same global namespace and may override or be overridden by application CSS.

Prefixed Footer (Proposed Fix)

Using the prefixed keyframe ease-kf-footer-float keeps the footer animation isolated from application styles while preserving the same visual behavior.

Heartbeat Animation

The same namespace collision can occur with @keyframes heartbeat. Prefixing it as ease-kf-footer-heartbeat prevents conflicts.
